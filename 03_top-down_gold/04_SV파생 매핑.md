# GOLD 파생→base 매핑 (GN_DW · 4단계)

derived 81개를 **물리 base measure(분자/분모)**, **소속 FACT**, **조인 conform 축**으로 분해한다. 계산식은 정본(`02·03 지표사전`) 원문을 그대로 인용하고, base 컬럼명은 `05_필드 인벤토리.md` 기준.

- **원칙(P2)**: derived는 GOLD 물리컬럼 금지 → **SV metric**. 본 문서는 그 SV metric이 참조할 base를 확정한다.
- **예외**: GA4 사전집계 비가산(평균세션시간 #98·이탈율 #108)은 재계산 불가 → FGA에 물리 적재(SV는 그 컬럼을 그대로 노출, 분자/분모 없음).
- 표기: `분자 / 분모`는 base measure 물리명. `FACT`는 base가 사는 팩트. `conform`은 비율 성립에 필요한 공통 집계축. ⚠️=합의/검토 플래그(§6).

## 6. 비판적 검토 / 보수적 결정 (오류 점검 결과)
> 매핑을 정본과 1:1 대조하며 발견한 쟁점. 추정으로 "고치지 않고" **정본 충돌은 그대로 노출 + 합의항목으로 격리**(보수적 원칙).

- **A. #46·#47 활동율 분자/분모 방향 충돌** — 정본 #45는 `월말활동/(연도초+누계개발)`인데 #46 신규활동율=`누계개발/활동`, #47 기존활동율=`활동/(누계개발+연도초)`. #46만 비율이 역방향(>1 가능). **임의 수정 금지** → 현업 확인 전까지 정본 그대로 매핑하고 §D-open 등록.
- **B. 신규/기존 분해의 스냅샷 귀속(#56·57·77·78)** — STOP/UNPAID/ACTIVE(건)은 FMM에 있으나, "전월말 신규활동회원건"처럼 **활동 스냅샷을 신규/기존으로 쪼개려면** 스냅샷 행에 NEW_EXISTING_FLAG가 보존돼야 함. FME 상태전이엔 있지만 FMM 월스냅샷 귀속은 미확정 → 5단계 확인(§5).
- **C. #80 미납회원 감소율의 base 단위 오류 방지** — 정본은 **명수**(월초/월말). 현 설계 FMM엔 미납(건)만 → 그대로 두면 분자가 틀림. **base 신설 후보로 격리**(§5), 임의로 건→명 대체 금지.
- **D. identity 의존 비율(#81·신32·33)** — GA 클릭/전환 "회원(명)"은 DIM_MEMBER_IDENTITY 매핑 성립이 전제. 또한 **클릭(명, distinct)** ≠ FAD `CLICKS`(행동 횟수). 분자/분모를 FGA distinct-member로 명시하고 identity 선행조건 표기.
- **E. 신#6 "이탈율" 명칭 vs 유지율 식** — 정본 계산식이 유지중/가입 = **유지율 형태**. 명칭(이탈율)과 상충 → 정본 식 우선 매핑하되 명칭/정의 합의 등록(신#7 유지율과 사실상 동일식).
- **F. 신#30 분모 "전체회원수"** — 활동회원인지 전체 universe인지 미정. base는 DIM_MEMBER/ACTIVE_MEMBERS 둘 다 가능 → 정의 확인 전 분모 확정 금지.
- **G. 광고비 출처 이원화(공#7·신#9)** — "광고비"가 FAD(AGENCY/GADS 실적) vs FBD(ERP 편성/모금성) 중 어느 것인지 지표별로 다름. 단가류는 비용 분모 출처를 **지표별로 명시**(기 분류 미해결 #6과 연결).
- **시계열 비교(#59·60·공, 신 누계류)** — 증감/누계는 **물리 저장 금지**(P7), SV time-intelligence. 누계는 base의 YTD 윈도우로 처리(새 컬럼 아님) — 본 문서 일관 적용 확인 완료.
- **GA4 비가산(#98·108)** — 분자/분모 없이 적재컬럼 직접 노출로 매핑. SV에서 SUM/AVG 재집계 금지(N) 표기 확인.

**검토 결론**: 정본 계산식과 충돌·미정인 7건(A~G)은 임의 수정하지 않고 합의 플래그로 격리했고, base 단위 오류 위험 2건(#80 명수·신32 클릭명)은 §5 신설/확인 후보로 분리했다. 그 외 72개 derived는 기존 base만으로 분자/분모가 모두 충족됨(추가 컬럼 불요).
