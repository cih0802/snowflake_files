# GOLD 스키마 설계 (GN_DW)

215 지표 + overview 필드를 담는 star schema. 원칙·용어는 `01_작업계획.md`.
- measure 60 → FACT / dimension 74 → DIM·degenerate / derived 81 → SV metric
- 물리 base measure ≳ 71 (logical 60 + GOAL_CNT + overview base: 콜·오픈·GA전환수·GA보조 + FEP 참여인원·FBD 예산 등). **logical measure 카운트는 60 불변.**

## 1. 설계 규칙 (요약)
- derived(율·LTV·ROI·구성비·증감)는 적재 안 함 → SV metric. **예외**: GA4가 사전집계로 직접 주는 비율·평균(평균세션시간#98·이탈율#108)은 재계산 불가 → FGA에 **비가산 measure** 적재(유형은 derived 유지).
- 월 grain=`MONTH_KEY`, 일 grain=`DATE_SK`. 회원 조인키=`MEMBER_DK`(불변).
- 미수령 원천도 컬럼 설계 + `_SOURCE_SYSTEM`. `(건)`=`SUM(금액)/10000` 다수 → 원금액 보존.

## 3. 팩트 (9 FACT)

### FMM — FACT_MEMBER_MONTHLY
- grain: MONTH_KEY × MEMBER_DK
- measure: 개발·중단·미납·활동·증액·감액 (건·명), 납입회비(원), 인바운드콜수·TS콜수, **UNPAID_MEMBERS(미납회원 명, 월말 스냅샷 — #80용)**
- FK: MONTH_KEY, MEMBER_DK, CAMPAIGN_SK, SPONSORSHIP_SK, PAYMENT_SK, REASON_SK
- snapshot/degen: 개발구분·금액대·기간대·납입개월수·후원기간
- note: FME(일 grain)의 월 롤업 스냅샷. **미납 판정(현업 확정 C-3)**: `TM_PM_MBRFEE_ACMSLT.PAY_STAT_CD IN ('F', NULL)`(S=성공/F=실패, NULL=수기처리 건이나 원래 F). 회원상태 MM010 미납1~5는 **#84 회원상태별 미납(건)** 분해용 dimension. **인바운드콜·TS콜은 CRM 원천에 없음 → 현업 별도 입고(C-8)**.

### 6-2. G-1·Q1 해소 → identity 지표 잠금해제
- **회원번호가 GA·CRM 공통 조인키**로 확정(현업 확인 G-1 ✅). GA `user_id` = CRM 회원번호(정규 7자리 / 일회성 `S`+8자리, VARCHAR·leading-zero 보존).
- **【회원번호 체계 명문화】 CRM 분리 관리 / GA 단일 통합**: CRM은 정식(FDRM)과 일시(ONCE) 회원을 **별도 테이블**로 분리 관리한다 — `TM_MM_FDRM_MBER_INFO`(번호 범위 `0000000`~`9999999`, 7자리, leading-zero 보존) · `TM_MM_ONCE_MBER_INFO`(번호 범위 `S00000000`~`S09999999`, `S`접두+8자리=9자). GA4 `user_id`는 이 두 유형을 **단일 필드에서 함께 표현** — `'S'` 접두 유무만으로 FDRM/ONCE 판별 가능. 두 원천이 동일 namespace의 회원번호를 공유하므로 직접 조인이 성립한다.
- **Q1(행매칭 실증) 해소**: 이 형식 스펙이 `GA4_IDENTITY`/`DIM_MEMBER_IDENTITY`가 `enabled=false`였던 사유(Q1)를 해소. 기존 분기 로직(`user_id ILIKE 'S%' → ONCE / else → FDRM`, `MBER_NO`/`ONCE_MBER_NO` 분리)이 스펙과 **정확히 일치(검증 완료)** → 두 모델 `enabled=true` 전환 가능.
- ⚠️ **활성화 시 동반 필수**: (a) D1-FGA — `FGA.IDENTITY_SK`가 실제값이 되므로 `unique_key`에 IDENTITY_SK 추가, (b) **DQ 확인** — GA export가 leading-zero·`S`접두를 보존해야 조인 성립(숫자화 시 매칭 실패).
- ⚠️ **[결론9] user_id 커버리지 4.2%**: identity 조인은 **구조적으로** 성립하나, GA4 `user_id` 채움률은 약 **4.2%**(전수 12,120/287,025·식별 1,290명·익명 95.8%). 따라서 #81·신#32·33 등 **회원단위 GA 지표는 부분 커버리지**이며 전수로 단정 금지. SV/BI에 커버리지 경고 노출, 익명(user_pseudo_id)은 세션 스티칭 없이는 회원 귀속 불가.
- 잠금해제 지표: **#81 미납서비스 전환율**, **신#32·33**(클릭/전환 회원명). 경로: `FGA.IDENTITY_SK → DIM_MEMBER_IDENTITY → 회원 미납 판정(C-3: PAY_STAT_CD F/NULL, 또는 회원상태 MM010 미납1~5 — ⑤에서 확정)`.
- **미납서비스 클릭회원(명)** = distinct(미납회원 ∩ GA 클릭회원). 소결정 2건 필요: ①"미납서비스 클릭"의 GA 이벤트/페이지 범위, ②"납입 전환" 판정(상태전이 vs 회비납입, 판정 기간).
