| 명/횟수 | `NUMBER(38,0)` | COUNT·노출·클릭 등 정수 카운트 |
| 원 | `NUMBER(18,2)` | 금액 SUM |
| 율/평균 | `NUMBER(9,4)` | 비가산 measure(N) — 재합산 금지 |
| 기간 | `NUMBER(9,2)` | 개월/년/일 |

**모든 테이블 공통 감사컬럼** (아래 표에서는 생략):
`DW_SOURCE_SYSTEM VARCHAR`, `DW_LOAD_TS TIMESTAMP_NTZ` (최초 적재시각, NOT NULL), `DW_UPDATE_TS TIMESTAMP_NTZ` (최종 적재시각), `DW_BATCH_ID VARCHAR` (= dbt `invocation_id` — 마지막으로 쓴 run UUID; 배치 단위 재적재 식별용).

> **merge 세만틱**: `dbt_project.yml` gold 경로에 `+merge_exclude_columns: ['DW_LOAD_TS']` 적용 — DW_LOAD_TS는 최초 INSERT 시각을 보존하고, DW_UPDATE_TS·DW_BATCH_ID는 run마다 갱신됨. DW_SOURCE_SYSTEM·DW_LOAD_TS만 `NOT NULL`.

**키 표기**: `PK`=기본키 · `BK`=업무키 · `DK`=불변키 · `FK→T`=T 참조 · `GRAIN`=grain 구성(degenerate 포함).

**DDL 제약 규칙 (DDL 단계·개발자 전달 CSV 공통)**
- **NULL**: `PK`·`GRAIN`·`DK`·`BK` 컬럼 = `NOT NULL`. 그 외 FK·measure·degen·snapshot = `NULL 허용`(방어적 — 미수령/미매칭 시 빈칸). 감사컬럼은 `DW_SOURCE_SYSTEM`·`DW_LOAD_TS`만 `NOT NULL`.
- **FK 강제성(중요)**: FK 제약은 **대상 DIM의 PK(=`*_SK`)에만** 설정 가능. 다음은 **조인용이나 FK 제약 불가**(Snowflake는 대상이 PK/UNIQUE여야 하며 아래는 비유일):
  - `MEMBER_DK` → DIM_MEMBER는 SCD2라 DK가 비유일. `ORG_DK` → DIM_ORG는 SCD1(유일)이나 FK는 관례상 PK(`ORG_SK`)에만 설정. 조인은 DK로, **FK 제약은 미설정**.
  - `MONTH_KEY` → DIM_DATE의 MONTH_KEY는 비유일·타입(6,0)≠DATE_SK(8,0). 월팩트 conform 조인은 **FK 미설정**(DIM_DATE 월초행 필터 또는 월 conform 뷰).
  - 따라서 `*_SK` FK만 `REFERENCES DIM_X(X_SK)`로 선언, DK/MONTH_KEY는 주석 처리.
- **팩트 복합 PK/UNIQUE**: 각 FACT의 grain 컬럼 집합이 논리 유일키. 단 FMM(MONTH×MEMBER)에 다치(多値) FK(CAMPAIGN/SPONSORSHIP/PAYMENT) 동반 등 **grain 정합 미확정 팩트는 PK/UNIQUE 미설정**(O8·확정 후 부여). 확정 전엔 ETL 멱등성으로 중복 방어.
- **VARCHAR 길이·NUMBER precision**: 본 인벤토리 타입은 제안값 → 정본 `06_지표용어사전`으로 DDL 단계 확정.

> 개발자 전달용 `gold 스키마 컬럼 인벤토리_*.csv`는 위 규칙을 컬럼으로 반영(`NULLABLE`, `FK_타깃` + 비강제 표기, 전 테이블 감사컬럼 4종 포함).

### 비판적 검토 (5단계 자체점검)
- **grain 위반 방지**: 04§5는 `UNPAID_MEMBERS`(명) 신설로 표기했으나, FMM grain=MONTH×MEMBER에서 "명수"는 measure가 아니라 행 조건 집계 → **snapshot flag**로 정정 적재. 이로써 logical measure 60 불변 유지(설계 §1 위배 없음).
- **추정 적재 금지(보수)**: #81 분자(납입전환 명)는 정의·identity 미해결 → **컬럼을 만들지 않음**. 임의 신설 시 SV가 잘못된 base를 참조할 위험이 더 큼.
- **denormalize 정당성**: `NEW_EXISTING_FLAG`를 FMM에 둔 것은 SCD2 DIM_MEMBER의 현재상태 조인으로는 **월 스냅샷 시점귀속**이 부정확하기 때문(스냅샷 팩트의 표준 패턴). 단 '신규' 정의 자체는 합의 전이므로 ⚠️ 유지.
- **단위 오류 차단**: FAD `CLICKS`(횟수) ↔ 서비스 클릭(명)을 명시 분리 — 신#32·33 분모를 FAD로 잘못 끌어오는 오류를 사전 차단.
- **신설 컬럼 영향 범위**: 추가 3컬럼(FMM 2 flag + 1 신규기존)은 전부 **snapshot/degen**, FK·grain·measure 카운트 불변 → 팩트×차원 참조표(03_설계 §4) 변경 없음.
- **잔여 위험**: 04§6의 A(#46 역방향)·E(신#6 명칭) 등 **계산식 충돌**은 base 컬럼 문제가 아니라 SV metric 정의 문제 → 인벤토리 변경 불요, SV 매핑 단계로 이월.
