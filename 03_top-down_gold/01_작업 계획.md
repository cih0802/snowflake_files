## 용어 (Glossary)
- **grain**: 팩트 1행의 최소 단위. 가장 먼저 확정. 혼재 시 과대집계.
- **FACT / DIM**: 숫자(measure) 저장 / 분류축(속성) 저장. FACT가 DIM을 FK 참조.
- **measure / dimension / derived**: 합산수치 / 분류속성 / 조합지표(SV에서 계산).
- **degenerate dim**: 차원이지만 DIM 테이블 없이 팩트 컬럼으로 저장(예 개발구분).
- **SK / BK / DK**: 대리키(ETL 일련번호) / 업무키(원천 식별값) / 불변키(상태변경에도 불변, 조인용).
- **conformed dim**: 여러 FACT 공용 표준 DIM(예 DIM_DATE).
- **SCD2**: 차원 변경 시 덮어쓰지 않고 새 버전행 추가(EFFECTIVE_FROM/TO, IS_CURRENT).
- **bridge**: 두 키를 1:N로 잇는 매핑(예 DIM_MEMBER_IDENTITY).
- **snapshot**: 특정 시점 상태값 적재(예 월말활동회원). 시변속성은 차원 아닌 스냅샷.
- **additivity**: A(가산) / S(준가산, 시점잔액) / N(비가산, 율·distinct).
- **basis**: measure 생성식. `(건)`=`SUM(금액)/10000`인 경우 다수 → 원금액 보존.
- **MONTH_KEY / DATE_SK**: 월=YYYYMM, 일=YYYYMMDD.
- **메달리온**: BRONZE(원천 1:1) → SILVER(정제통합) → GOLD(분석 star schema).
- **Semantic View(SV)**: GOLD 위 의미계층. derived를 metric으로 정의, Cortex Analyst/Agent 사용.
- **FQN**: `DATABASE.SCHEMA.OBJECT` (예 `GN_DW.GOLD.FACT_MEMBER_MONTHLY`).
