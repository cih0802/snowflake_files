## E. GA4 행동지표 산출 규칙

### G-2. 이탈율·평균세션시간 산출규칙 — 🅿️ 우리 기본안 제시(현업 승인 요청)
- **🅿️ 제안(GA4 콘솔 공식 채택)**: 이탈율 = `1 − DIV0(ENGAGED_SESSIONS, SESSION_CNT)` · 평균세션시간 = `SUM(engagement_time_msec)/1000 / DISTINCT 세션`. 승인 시 FGA `BOUNCE_RATE`·`AVG_SESSION_DURATION`(현 NULL) 즉시 채움.
- **📗 확신도**: ✅ "이탈율=1−참여율"·참여세션 3조건(10초/전환/2뷰)은 GA4 공식(확정). ✅ `SESSION_ENGAGED`는 GA4 native 출력이라 직접 사용=콘솔과 정의상 동일(샘플 검증: engaged=0 세션 99.6%가 3조건 미충족). ⚠️ '평균세션시간'을 engagement 기반으로 본 건 부분추측 → 이용시간 항목과 함께 확정.
- **⚠️ 비가산 주의**: 이탈율·세션시간은 페이지·기기별 단순 합/평균 금지(세션수 가중 필요), SV·BI 재합산 금지. (GA4 콘솔도 동일)
- **📖근거**: [회원 §3-2]·[공통 #108·#98] · 원천 `GA4_EVENT.SESSION_ENGAGED`·`ENGAGEMENT_TIME_MSEC`.
- **✍️현업 승인란**: ✅ **승인(현업 회신)** — "우리 제안대로 GA4 콘솔 기준으로 진행". 📌 추후 현업이 계산식을 전달하면 수정 가능(코드에 메모 유지). → FGA `BOUNCE_RATE`·`AVG_SESSION_DURATION` 구현 착수.

### 이용시간. '이용시간'의 정의 — 🅿️ 우리 기본안 제시 (engaged time 권장)
- **🅿️ 제안**: 이용시간 = engaged time(`AVG_ENGAGEMENT_TIME_PER_SESSION`, 이미 FGA 산출) 채택. G-2와 일치.
- **🔎 데이터 근거(샘플)**: 참여세션 평균 engaged=9.46초 vs wall-clock=84.29초(8.91배)지만 **65.9%가 단일 이벤트 세션(wall-clock=0)**이라 wall-clock 평균은 대표성 없음(zero-inflation). engaged는 중앙값 3.98초로 안정적. wall-clock은 [회원 §3-2] 4개 세션시간 필드 중 어느 게 '이용시간'인지 확정 후 필요 시 `(MAX-MIN EVENT_TIMESTAMP)/1e6`로 추가 가능.
- **📖근거**: [회원 §3-2] 세션시간류 4필드 · [공통 #98]. ※ G-2와 묶어 결정.
- **✍️현업 승인란**: ✅ **승인(현업 회신)** — 제안대로 engaged time으로 진행. 📌 추후 현업 계산식 전달 시 wall-clock 등으로 수정 가능(메모 유지). G-2와 함께 확정.

### O2. DIM_DEVICE PC/M/APP 판정 — ✅ 규칙 확정 (GA 전문가 자문 + 데이터 검증)
- **✅ 확정 규칙** (`PLATFORM`만으론 PC/모바일웹 구분 불가 → `DEVICE_CATEGORY` 병용):
  - **PC** = `PLATFORM='WEB'` AND `DEVICE_CATEGORY='desktop'`
  - **M** = `PLATFORM='WEB'` AND `DEVICE_CATEGORY IN ('mobile','tablet')`
  - **APP** = `PLATFORM IN ('IOS','ANDROID')`
- **근거**: GA 전문가(GA4 platform=web/iOS/Android만 → device.category 필수) + 데이터 검증(샘플 전량 WEB, PC·M 모두 platform=WEB 확인). silver `GA4_EVENT.DEVICE_TYPE`가 **이미 desktop→PC·mobile/tablet→M 매핑 완료** → **APP 분기만 추가**하면 됨. **잔여(현업 아님)**: 앱데이터 입고 검증·웹앱 GA4 속성구조 확인.
- **📖근거**: [회원 §1-1]·[공통 #14] · 원천 `GA4_EVENT.PLATFORM`·`DEVICE_CATEGORY` → `DIM_DEVICE`(PC/M/APP).

### SF-biz. 익명 트래픽 미귀속 — 🛠️ 설계로 흡수 (원본 보존 + 식별률 고지, 현업 차단 아님)
- **성격**: 로그인 없는 세션은 원본에 회원번호 부재 → 복구 불가(물리 한계). 수용/거부 결정이 아니라 **고지 + 설계 흡수**.

- **📖근거**: [공통 #112·#99/#101] · 원천 `GA4_EVENT.USER_PSEUDO_ID`↔`USER_ID`→파생 · (session-fill 채택 자체 = 33 §B SF).
