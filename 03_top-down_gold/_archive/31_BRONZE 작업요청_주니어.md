## T1. 적재 메타 컬럼 추가 — 우선순위 ★★★

**배경**: CRM 테이블은 공통으로 `_LOAD_DT`·`_BATCH_ID`를 갖고 있는데, 이번에 만든 ERP·AGENCY 테이블엔 없습니다. 증분 적재·재적재 추적을 위해 통일해 주세요.

**대상 테이블 (4개)**
- `GN_DW.BRONZE_ERP.BDGT_ACMSLT_LEDGER`
- `GN_DW.BRONZE_AGENCY.DGT_AD_CMPGN_DTLS`
- `GN_DW.BRONZE_AGENCY.REBRDC_AD_CMPGN_DTLS`
- `GN_DW.BRONZE_AGENCY.VIDEO_AD_CMPGN_DTLS`

**할 일**: 각 테이블에 아래 2개 컬럼 추가 (CRM과 동일 타입).

```sql
ALTER TABLE GN_DW.BRONZE_ERP.BDGT_ACMSLT_LEDGER
  ADD COLUMN _LOAD_DT TIMESTAMP_NTZ(9) COMMENT '적재일시',
             _BATCH_ID VARCHAR(50)     COMMENT '적재배치ID';
-- 나머지 3개 AGENCY 테이블도 동일하게 반복
```

**완료 기준**: 4개 테이블 모두 `_LOAD_DT`·`_BATCH_ID` 존재, 적재 시 값 채워짐.

## T2. AGENCY 광고 테이블에 `_SOURCE_SYSTEM` 추가 — 우선순위 ★★★ (A-2)

**배경**: 광고비·노출·클릭·전환은 **대행사**와 **Google Ads** 두 곳에서 옵니다. 지금은 섞이면 구분이 안 돼 매체별 개발단가/ROI 계산이 틀어집니다.

**대상**: AGENCY 광고 3테이블 (T1과 동일 3개).

**할 일**
```sql
ALTER TABLE GN_DW.BRONZE_AGENCY.DGT_AD_CMPGN_DTLS
  ADD COLUMN _SOURCE_SYSTEM VARCHAR(30) COMMENT '데이터 출처 (AGENCY / GOOGLE_ADS)';
-- 재송출·영상 테이블도 동일
```
- 적재 시 대행사분은 `'AGENCY'`, Google Ads분은 `'GOOGLE_ADS'`로 채워 주세요.

**완료 기준**: 모든 광고 행에 `_SOURCE_SYSTEM` 값이 채워져 있고, 두 출처가 구분됨.

## T3. 어드민 3원천 신규 적재 — 우선순위 ★★ (A-5·A-6·A-10)

**배경**: 아래 3종은 어드민 화면에서 수집되는 데이터로, 현재 어느 테이블에도 없습니다. 원천(어드민 export 파일/DB) 위치부터 같이 확인이 필요합니다.

| 필요 데이터 | 최소 컬럼 | 이슈 |
|---|---|---|
| 앱푸시 | 발송일자, 캠페인/메시지 식별, **발송건수**, **성공건수** | A-5 |
| 이벤트(행사) 조회수 | 행사 식별, 일자, **조회수** | A-6 |
| 행사 정보 | 행사 식별, **시작일·종료일**, **참여경로**, **참여채널** | A-10 |

**할 일**
1. 먼저 어드민 원천 위치·추출 방법을 확인해 회신 (파일 export? DB 직접?).
2. 확인되면 `BRONZE_AGENCY`(또는 별도 스키마)에 신규 테이블 생성 후 적재.

**완료 기준**: 3종 데이터가 BRONZE 테이블로 적재되고, 위 "최소 컬럼"이 모두 존재.

> 참고: `DGT_AD_CMPGN_DTLS.READ_CNT`(조회수)는 **광고 조회수**라서 A-6(행사 조회수)와는 다릅니다. 혼동 주의.

## T4. ERP 캠페인/매체 연결키 확인 — 우선순위 ★★ (E-1·E-4)

**배경**: `BDGT_ACMSLT_LEDGER`는 예산 계정 계층(장/관/항/목/세목/세세목·재원)까지만 있고, **어떤 캠페인·매체에 쓴 비용인지 연결할 키가 없습니다.** 이게 없으면 캠페인별 개발단가/ROI를 낼 수 없습니다.

**할 일 (순서대로)**
1. ERP 원천에 캠페인 코드·매체 코드(또는 그에 준하는 식별자)가 있는지 확인.
2. **있으면**: 해당 컬럼을 테이블에 추가해 함께 적재.
3. **없으면**: 어느 단위(예: 예산단위/재원)까지 연결 가능한지 회신 → 현업과 집계 기준 합의.

**완료 기준**: 캠페인/매체 연결키 적재 완료, 또는 "불가 + 대체 단위" 회신.

---

## T5. 송출일 ≠ 실적일 분리 보존 — 우선순위 ★ (A-8)

**배경**: 재송출·영상 광고는 **송출된 날**과 **실적이 잡히는 날**이 다를 수 있습니다. 지금 테이블엔 날짜가 1종(`DATE`/`BRDC_DATE`)만 있어 구분이 안 됩니다.

**대상**: `REBRDC_AD_CMPGN_DTLS`, `VIDEO_AD_CMPGN_DTLS`

**할 일**: 원천에 송출일·실적일이 별도로 있으면 **둘 다** 컬럼으로 보존(예: `BRDC_DATE` 송출일 + `PERF_DATE` 실적일). 하나뿐이면 어느 쪽인지 회신.

**완료 기준**: 송출일/실적일 구분 보존 또는 "1종만 존재 + 어느 쪽인지" 회신.
