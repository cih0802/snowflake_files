후속작업자는 아래 순서만 따르면 워크스페이스 파일만으로 작업을 이어갈 수 있습니다.

1. **본 문서(`파일명세_20260703.md`)** — 프로젝트 전체 지도. 어떤 파일이 무엇인지.
2. **`DBT_구현_예견이슈_20260703.md`** — 🔴필독. 이슈 A~S(GA4 증분 함정·TRUNCATE 규칙·unknown 멤버·GA4_IDENTITY 매칭0 등). 여기 안 보면 지뢰 밟음.
3. **`GOLD_파이프라인_dbt_작업가이드_20260703.md`** — SILVER→GOLD 적재 규칙(대리키·SCD2·미수령 제외·적재순서).
4. 구조 정본(참조용): `04_silver_design/SILVER_DDL_20260702.sql`, `03_top-down_gold/06_DDL.sql`, `03_top-down_gold/08_silver의존.md`(lineage), `04_silver_design/GA4_*` 2건, `99_provided_definition/09_bronze_crm_ddl.sql`(Bronze 물리타입 정본).
5. 환경 이전 시: `MIGRATION_GUIDE.md`.

**현재 상태 한 줄 요약**: SILVER 26 + GOLD 19 모델 작성 완료. 실데이터 검증분 = `CRM_MEMBER`(2,000) + GA4 4개 + gold_ready 6개(29 tests PASS). 나머지는 Bronze 입고/동일모집단 데이터 대기(compile만 통과).

**다음 작업 진입점**:
- Bronze CRM 입고 시 → 이슈문서 §F·N + GOLD가이드 §2 보고 `dbt run --select silver.crm` → `tag:gold_pending`.
- GA 회원분석 필요 시 → 이슈문서 §S(매칭0 해소조건) 먼저 확인.
- 실행/재적재 규칙 → 본 문서 맨 아래 "실행 요약" + 이슈문서 §L(TRUNCATE).

## 실행 요약 (안전 커맨드)

```bash
# 실데이터 검증 세트

dbt run --select silver.ga4 --full-refresh     # GA4 최초적재
dbt run --select tag:gold_ready                # GOLD 실적재 6개

# Bronze 입고 후
dbt run --select silver.crm                    # CRM 20개
dbt run --select tag:gold_pending              # GOLD 스캐폴드 13개

# ⚠️ 무선택 dbt run 금지 — 미입고 Bronze 참조로 실패
```
