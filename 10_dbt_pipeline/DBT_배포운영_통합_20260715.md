## 2. 운영 계약 — 온디맨드 `build` (★현행)

- BRONZE 원천 = 정기 갱신·**주기 불규칙** → 고정 CRON TASK는 안티패턴(보류).
- **현행**: 적재 완료 후 아래 한 줄 호출. **`run` 아닌 `build`**(run+test) — `run`은 0행도 SUCCESS라 조용한 회귀 미검출.
```sql
-- 전체 재정제
EXECUTE DBT PROJECT GN_DW.OPS.DW_PIPELINE ARGS='build';
-- 부분: GA4 샤드 입고 시(하류 XREF 포함) / CRM만
EXECUTE DBT PROJECT GN_DW.OPS.DW_PIPELINE ARGS='build --select silver.ga4+';
EXECUTE DBT PROJECT GN_DW.OPS.DW_PIPELINE ARGS='build --select silver.crm';
```
- ⚠️ **full `build`는 여전히 실패(의도적)** — `EVENT_KEY→CRM_EVENT` 고아 263,611(참여 23%, 이슈 E) 때문. **GOLD-only(`--select path:models/gold`)만 green.** (이슈 원장 문서50 BLOCKING-2)
- **최종형(보류)**: BRONZE에 STREAM + `WHEN SYSTEM$STREAM_HAS_DATA(...)` 트리거 TASK. 적재 메커니즘 확정 후 구현.

### 운영 명령어
| 작업 | SQL |
|---|---|
| 프로젝트/버전 확인 | `SHOW DBT PROJECTS IN SCHEMA GN_DW.OPS;` / `SHOW VERSIONS IN DBT PROJECT ...` |
| GOLD만 적재·검증 | `ARGS='build --select path:models/gold'` (SILVER 테스트 게이트 우회) |
| 개별 모델 재실행 | `ARGS='build --select <model>'` |
| 프로젝트 삭제 | `DROP DBT PROJECT GN_DW.OPS.DW_PIPELINE;` |

## 3. 불변 운영 규칙 (재발방지 — 반드시 준수)

| # | 규칙 | 근거(사고) |
|---|---|---|
| R1 | **로직 변경(SK 산식·grain·조인) = 대상 테이블 `TRUNCATE` 후 재적재** | `full_refresh:false`라 merge가 스테일 행을 안 지움 → 실측 12,202→13,445 누적 |
| R2 | **`run` 금지, `build` 사용** | GA4 매크로 0행 회귀가 `run`에선 SUCCESS로 통과(조용한 손실) |
| R3 | **BI/팩트-회원 조인 시 `DIM_MEMBER.IS_CURRENT=TRUE` 필터** | 팩트가 `MEMBER_DK` 자연키 조인 → SCD2 다버전 시 팬아웃 |
| R4 | 구조 소유주 = DDL(`08_SILVER_테이블DDL`·`06_DDL`), dbt는 데이터만 갱신(`full_refresh:false` + pre-hook TRUNCATE + append) | DDL이 타입·제약·FK 소유. fact `table` 폐기(구조 덮어써 FK 드롭) |
| R5 | 스캐폴드 차원엔 unknown(SK=0) 멤버 필수 | 없으면 센티넬 팩트가 inner join서 전량 유실 |

## 4. 잔존 이슈 (미해소 — 이슈 원장 연계)

> 이미 해소된 예견이슈(GA4 매크로 대소문자·DIM_DATE 고정캘린더·unknown멤버·GA4회귀)는 아카이브. 아래는 **실측상 아직 살아있는 것**.

| ID | 이슈 | 상태 | 원장 연계 |
|---|---|---|---|
| D | GA4_EVENT 복합PK 유일성 = 단일샤드만 검증 | 🔴 다중샤드 미검증 | 문서40 G-5(전기간 입고 시 아래 쿼리로 검증) |
| E | DEVICE_TYPE(PC/M/APP) 파생 휴리스틱 미검증 | 🟡 category 도메인 미확정 | 문서40 G-5 / 문서30 O2 |
| S | GA4↔CRM 신원 행매칭(identity) | 🔴 샘플 분리로 매칭 실증 대기 | 문서30 ID-활성 / 문서40 G-1 |
| G | `CRM_CODE` 라벨(`_NM`) 백필 방식 미결(인라인 vs GOLD 조인) | 🟡 설계결정 대기 | 문서30 설계결정 |
| M | DIM_GA_SOURCE grain 축소(4키→2키, content/term 부재) | 🟡 수용, GOLD 계약 대조 필요 | — |
| — | GOLD 미작성 4종·WIDE VIEW | ✅ 순서9-C/9-D 해소(GOLD 4종 작성·WIDE 8/9 배포). 잔여 = FACT_TARGET_BIZ+WIDE_TARGET_BIZ(E-6) | §5·§7, 문서30 §6 |

## 5. GOLD 미작성 4종 — 실제 스코프 [순서9-C 재평가]

> ⚠️ 이전 "4개 즉시 가능" 은 부정확. 실측 결과 **2개만 clean, 1개 부분, 1개 외부 블로커 의존**.

| GOLD 대상 | SILVER 소스 | 상태 | 블로커 |
|---|---|---|---|
| `DIM_BUDGET_ITEM` | `ERP_BUDGET_ITEM`(2,040) | 🟢 즉시 가능 | 없음 |
| `DIM_AD_CREATIVE` | `AGENCY_AD_CREATIVE`(8,473) | 🟡 가능(컬럼 재매핑 가정) | 원천별 산재 컬럼 매핑 확인 |
| `FACT_BUDGET` | `ERP_BUDGET`(24,480) | 🟡 편성/집행만 | 모금성비용(E-1)·광고비(E-4) 원천부재 → NULL |
| `FACT_AD_PERFORMANCE` | AGENCY+GA4 | 🔴 스캐폴드만 | 캠페인 이름매칭(Q10)·`_SOURCE_SYSTEM`(A-2/Q9)·GA전환정의(O5) |

- 공통: 구조 소유주 = `03_top-down_gold/06_DDL.sql`. dbt `+full_refresh:false`. 매크로 `gold_sk`·`gold_meta`·`month_key`·`date_sk`.
- `FACT_TARGET_BIZ`(ERP_BIZ_TARGET 0행·E-6)·`DIM_MEMBER_IDENTITY`(enabled=false)는 **작성 보류**.
- WIDE VIEW: **순서9-D 에서 8종 dbt view 모델화 완료**(`models/gold/wide/`, `ref()`·`materialized:view`, COMMENT=모델 post_hook, 8뷰 compile SUCCESS). 9번째 `WIDE_TARGET_BIZ` 는 `FACT_TARGET_BIZ` 모델 부재(E-6)로 보류 → fact 저작 시 동시 추가. 상세: 문서50 §BLOCKING-4.
