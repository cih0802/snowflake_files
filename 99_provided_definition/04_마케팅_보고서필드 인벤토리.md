> **이 문서는 정본(authoritative)이다.** 현업이 전달한 마케팅 보고서의 *필드 구성과 그 원천(source-of-record)* 을
> 빠짐없이 담고 있다. LLM 가독을 위해 표로 구조화했을 뿐, **내용은 현업 전달 그대로**이며 이 폴더가 권위있는 원본이다.
>
> **담는 것**: 마케팅 분석 보고서들이 *어떤 필드를 소비하고, 그 필드가 어느 원천(CRM/ERP/대행사/GA)에서 오는지*.
> **담지 않는 것(다른 정본 참조)**: 지표 계산식 → `02_지표사전_공통.md`·`03_지표사전_신규.md` / 물리 테이블 → `../03_top-down_gold/03_설계.md`.
> 이 둘은 권위 부재가 아니라 **역할 분담**이다(필드·원천 정본 = 본 문서).
>
> **LLM 사용 지침**: (1) 필드·원천의 정본으로 신뢰해 사용. 계산식만 지표사전에서 조회. (2) `데이터 원천`은
> 아래 정규화 범례로 표준 코드(CRM/ERP/GA4/AGENCY/GADS)에 매핑해 해석. (3) `(상속)`·`(미정)`은 원본 병합셀/공란
> 표기로, 적재 전 원천팀 확인이 필요한 데이터-품질 주의값(문서 권위와 무관).

## 원천 정규화 범례 (원본 보존 + 표준 코드 병기)

| 원본 표기 | 표준 원천 코드 | 비고 |
|---|---|---|
| CRM | `CRM` | 회원관리시스템 |
| ERP | `ERP` | 예산/집행(미수령 원천) |
| 대행사 자료(일별레포트) | `AGENCY` | 광고대행사 일별 레포트(미수령 원천) |
| GA | `GA4` | Google Analytics 4 |
| Google Ads | `GADS` | Google Ads |
| 대행사 자료(일별레포트) / Google Ads | `AGENCY+GADS` | 복수 원천 |
| 대행사 자료(일별레포트) / GA | `AGENCY+GA4` | 복수 원천 |
| - | `N/A` | 코드성/구분값(원천 비해당) |
| (상속) | *(추론)* | 원본 병합셀 — 직전 명시 원천 상속(검증 필요) |
| (미정) | *(미확인)* | 원본 공란 — 원천 확인 필요(예: 부서명은 CRM 가능성 높음) |

> **Caveat**: 위 `(상속)`/`(미정)` 항목과 데이터 TYPE 일부는 원본 CSV의 병합셀·공란을 기계적으로 복원한 것이라
> 실제 적재 전 원천팀 확인이 필요하다. (필드·원천 자체는 본 정본을 따르고, GOLD 컬럼→원천 최종 lineage는 `../03_top-down_gold/07_silver의존.md` 예정.)

## 부록: 표기 규칙

- **(상속)**: 원본 시트에서 병합셀로 인해 비어 있던 값. 해당 섹션의 직전 명시된 데이터 원천을 따른다.
- **(미정)**: 원본에 데이터 원천/타입이 비어 있어 추가 확인이 필요한 항목.
- **데이터 TYPE 종류**: 문자 / 숫자 / 숫자(회계) / 백분율 / 날짜 / 시간 / 문자+숫자 / 숫자+문자
- **주요 데이터 원천**: CRM, ERP, 대행사 자료(일별레포트), GA(Google Analytics), Google Ads
