## 3. 사용 규칙 (LLM 지침)
1. **계산식** → `02_지표사전_공통.md`/`03_지표사전_신규.md` (정본)
2. **필드의 원천** → 인벤토리 2종에서 찾고 §2 표준코드로 해석 (정본)
3. **용어·데이터타입** → `06_지표용어사전 20260624.md` (`지표구분`으로 범위 한정)
4. **CRM 물리 테이블** → `BRONZE_CRM 테이블 정보.MD`와 대조
5. **GOLD 컬럼의 최종 lineage** → `../03_top-down_gold/07_silver의존.md`(예정)가 우선. 인벤토리의 `(상속)`/`(미정)`은 추론값.
6. **미수령 원천(ERP/AGENCY/GA4/GADS/ADMIN)** → 인벤토리 원천을 출발점으로 삼되 원천팀 확인 전까지 잠정.

## 4. 검증 필요(open)
- 마케팅 `부서명` 원천 공란 → CRM 추정(지표사전 #116 '부서') 확인.
- 회원 `앱참여수치분석보고`의 CRM/GA/ADMIN 원천 혼재 → 컬럼별 원천 재확정.
- 회원 `참여콘텐츠분석보고(온라인이벤트)` 등 **중복 필드** → 적재 전 정리.
- 용어사전 카운트(공통167·신규50·오버뷰300) ≠ 지표사전(162·53) → 항목 1:1 매핑 미검증.
- 인벤토리 데이터 TYPE은 보고서 표기 기준 → GOLD 물리 타입과 별도 매핑.
