> **이 문서는 정본(authoritative)이다.** 현업이 전달한 회원 분석 보고서의 *필드 구성과 그 원천(source-of-record)* 을
> 빠짐없이 담고 있다. LLM 가독을 위해 표로 구조화했을 뿐, **내용은 현업 전달 그대로**이며 이 폴더가 권위있는 원본이다.
>
> **담는 것**: 회원 분석 보고서들이 *어떤 필드를 소비하고, 그 필드가 어느 원천(CRM/GA/어드민)에서 오는지*.
> **담지 않는 것(다른 정본 참조)**: 지표 계산식 → `02_지표사전_공통.md`·`03_지표사전_신규.md` / 물리 테이블 → `../03_top-down_gold/03_설계.md`.
> CRM 필드는 `BRONZE_CRM 테이블 정보.MD`(40테이블/789컬럼)와 대조해 실제 원천 테이블을 확정한다.
>
> **LLM 사용 지침**: (1) 필드·원천의 정본으로 신뢰해 사용. 계산식만 지표사전에서 조회. (2) `데이터 원천`은
> 아래 정규화 범례로 표준 코드에 매핑해 해석. (3) 보고서별 CRM 표와 GA 표는 **행이 매칭되지 않는 독립 목록**임.

## 원천 정규화 범례 (원본 보존 + 표준 코드 병기)

| 원본 표기 | 표준 원천 코드 | 비고 |
|---|---|---|
| CRM | `CRM` | 회원관리시스템 |
| CRM(UMS) | `CRM_UMS` | CRM 메시지발송(UMS) 영역 |
| CRM/계산 | `CRM_CALC` | CRM 기반 계산식 산출값 |
| GA | `GA4` | Google Analytics 4 |
| 어드민 > 이벤트목록 | `ADMIN` | 어드민 화면 수집 |
| 어드민 > 모바일앱 > 푸시발송목록 | `ADMIN` | 어드민 화면 수집 |

> **Caveat**: 데이터 TYPE 일부와 중복 필드는 원본 CSV를 충실히 보존한 것이다. 특히 일부 보고서(예: 앱참여수치분석보고)는
> 원본 CRM 컬럼 안에 GA·어드민 원천이 혼재되어 있어 `데이터 원천` 값을 그대로 따라야 한다.
> 정본 원천 확정은 본 문서(필드·원천)와 `BRONZE_CRM 테이블 정보.MD`(물리 테이블)를 따르고, GOLD 컬럼→원천 최종 lineage는 `../03_top-down_gold/07_silver의존.md` 예정이다.

## 문서 구조

- 큰 영역 **3개**: `1. 개발현황`, `2. 회원특성`, `3. 서비스`
- 각 영역은 보고서(**분석 구분**)별로 나뉜다.
- 원본은 보고서마다 **좌측 CRM 필드 목록**과 **우측 GA 필드 목록**이 나란히 배치된 2단 구조다.
  - 두 목록은 **서로 행이 매칭되지 않는 독립 목록**이다. 따라서 본 문서에서는 보고서별로 `CRM 필드` 표와 `GA 필드` 표로 분리한다.
  - GA 필드가 없는 보고서는 CRM 표만 둔다.

## 부록: 표기 규칙

- **CRM 필드 / GA 필드**: 원본의 좌(CRM)·우(GA) 2단 독립 목록을 분리 표기한 것. 두 표는 행 단위로 매칭되지 않음.
- **CRM/계산**: CRM 원본 데이터를 기반으로 계산식으로 산출되는 값.
- **CRM(UMS)**: CRM의 UMS(메시지 발송 시스템) 영역에서 가져오는 값.
- **어드민 > ...**: 별도 어드민 화면 경로에서 수집되는 값.
- ***(중복)***: 원본 시트에 동일 필드명이 중복 입력된 항목. 충실성을 위해 보존했으며 실제 적재 시 정리 필요.
- **데이터 TYPE 종류**: 문자 / 숫자 / 숫자(회계) / 백분율 / 백분율(계산식) / 날짜 / 시간(분,초) / 문자+숫자 / 숫자+문자 / URL(회원번호포함) / URL(UTM, 회원번호숫자포함)
- **주요 데이터 원천**: CRM, GA(Google Analytics), 어드민, CRM(UMS)
