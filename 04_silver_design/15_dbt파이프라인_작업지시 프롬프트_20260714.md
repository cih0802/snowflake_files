## 🔴 GOLD 소비 계약 (신원브리지, 순서 8에서 반드시 준수)

`IDENTITY_MEMBER_XREF` 설계의 핵심 소비규칙 — 파이프라인이 이 값들을 GOLD로 흘릴 때 강제.

| # | 규칙 | 위반 시 결과 |
|---|---|---|
| **C1** | `FACT_GA_BEHAVIOR` → XREF는 **LEFT JOIN** 필수 | INNER JOIN이면 GA 이벤트 95% silent 소실(익명 pseudo 26,492개 미매칭) |
| **C2** | `DIM_MEMBER_IDENTITY` 구축 시 **MEMBER_DK로 DISTINCT** 필수 | pseudo 1,348 ≠ member 1,274 → 중복 계상 |
| **C3** | 회원차원 구축은 `WHERE MATCH_METHOD='MEMBER_ID_EXACT'` 필터 필수 | `DIM_MEMBER_IDENTITY.MEMBER_DK` NOT NULL 제약 위반 |
| **C4** | MEMBER_DK 길이(9 ≤ VARCHAR10)·공백 0 PoC 실측 | 전기간 재적재 시 재검증 필수 |

## 🎯 순서 7: dbt 파이프라인 일괄 구성 — 작업 범위

**목적**: BRONZE → SILVER 정제 적재를 자동화·스케줄링. GOLD 적재는 **포함 안 함(순서 8)**.  
**원칙 C(master §7)**: dbt 모델(테이블 정의·적재 로직) 구성은 OK. 오케스트레이션/스케줄링(Task·dbt job·증분 트리거)은 **전체 SILVER 안정화 후 일괄**이므로 지금이 그 시점.

### 7-A. 확인 및 현황 파악
1. `/workspace/10_dbt_pipeline/` 구조 전체 탐색(dbt_project.yml, profiles.yml, models/, macros/).
2. 현재 존재하는 dbt 모델 목록 확인 → SILVER 32객체와 대조.
3. `ga4_union_shards` 매크로 현재 내용 확인.

### 7-B. dbt 모델 정비 (SILVER 32객체)
- 09_SILVER_적재쿼리 STEP 3~7의 INSERT OVERWRITE를 dbt `materialized='incremental'` 또는 `table`로 변환.
- GA4 5객체: `ga4_union_shards` 매크로 참조, `FROM` 절을 매크로로 교체(전기간 샤드 입고 후 멱등).
- `IDENTITY_MEMBER_XREF`: GA4_IDENTITY 적재 후 실행(의존성 순서 = GA4_IDENTITY → XREF).
- CRM_BIZ_TARGET(0행 스키마-only, 원천=CRM 확정 2026-07-20·구 ERP_BIZ_TARGET): 모델 생성하되 소스 미입고 시 빈 상태 유지.
- AGENCY_COST: SILVER 모델 제외(GOLD 이관 결정).

### 7-C. 의존성 DAG 설계
```
BRONZE_CRM    → CRM_* (21모델, 독립) + CRM_BIZ_TARGET(신규·사업목표, 입고 대기)
BRONZE_GA4    → GA4_EVENT → GA4_IDENTITY → IDENTITY_MEMBER_XREF
              → GA4_TRAFFIC_SOURCE, GA4_EVENT_DIM, GA4_DEVICE
BRONZE_ERP    → ERP_BUDGET_ITEM → ERP_BUDGET (FK: BUDGET_ITEM_DK)
BRONZE_AGENCY → AGENCY_AD_CREATIVE, AGENCY_AD_PERFORMANCE
```
**⚠️ IDENTITY_MEMBER_XREF는 GA4_IDENTITY AND CRM_MEMBER 모두 완료 후 실행.**

### 7-D. 스케줄·트리거 구성
- Snowflake Task 또는 dbt Cloud/Snowflake Job으로 스케줄.
- GA4: 신규 샤드 `events_YYYYMMDD` 입고 트리거 → ga4_union_shards 실행.
- CRM/ERP/AGENCY: 일배치 권장(변경분 체크: STREAM 또는 MAX(DW_LOAD_TS) 비교).
- 실행순서: CRM·ERP·AGENCY 병렬 → GA4(샤드 의존) → IDENTITY_MEMBER_XREF.

### 7-E. 멱등성 보장
- 모든 SILVER 모델 = `INSERT OVERWRITE`(09 정본 패턴) 유지 → `full_refresh` 동치.
- DW_LOAD_TS = `CURRENT_TIMESTAMP()`(재실행 시 갱신) — 프로젝트 관례. 변경 금지.

## ⚠️ 작업 가드레일 (위반 금지)

1. **SILVER에서 집계 금지** — GROUP BY 집계는 GOLD/SV 전용(단, `CRM_SEND_RESULT` 채널 통합은 기검토된 예외).
2. **SILVER → SILVER 참조 금지** — 단방향(BRONZE→SILVER). 유일 예외=`IDENTITY_MEMBER_XREF`만.
3. **GOLD DDL 미배포** — `GN_DW.GOLD` 스키마 미존재. 순서 7에서 GOLD 테이블 생성·적재 금지.
4. **GA4 전기간 샤드 미입고** — 현재 1일 샤드만. `ga4_union_shards` 매크로로 래핑하되, 데이터는 PoC 상태 유지(커넥터 소관).
5. **CRM_BIZ_TARGET**(구 ERP_BIZ_TARGET) — 0행(원천=CRM 확정, CRM 신규 목표 테이블 입고 대기 E-6). dbt 모델 생성은 OK, 데이터 없음이 정상. 단위=건(#152~155).
6. **텍스트 날짜 파싱 금지** — 날짜는 `TO_DATE()`·`TO_TIMESTAMP_NTZ()` 파생, `CAST('20260101' AS DATE)` 형태 금지.
