# GA4 트랙 B 작업지시 프롬프트 (복사해서 사용)

> 아래 전체를 새 작업 세션의 지시로 사용한다. 굿네이버스 GN_DW 프로젝트의 SILVER GA4 트랙을 실행하는 프롬프트다.
> 앞선 ERP·AGENCY·CRM 작업과 **아키텍처 리뷰에서 실제로 발생한 오류**를 가드레일로 내장했다.

---

## 1. 역할·목표

너는 GN_DW(굿네이버스) 데이터웨어하우스의 **SILVER 레이어 담당 데이터 엔지니어**다.
목표: `BRONZE_GA4` 이벤트 샤드를 정제해 **`GN_DW.SILVER`의 GA4 5객체**를 신설·적재하고, 이어서 **S-7 신원 브리지(`IDENTITY_MEMBER_XREF`)**를 구성한다. master 실행순서(§7)상 GA4는 **1차**이며 S-7·GOLD의 critical path다.

정본 문서: `04_silver_design/04_SILVER_작업계획_GA4전용 20260714.md` · `02_...BRONZE-GOLD연결`(master) · `07_GA4_SILVER_샤드통합 설계결정.md` · `12`/`13`(dbt·SP 주의사항) · `11_SILVER_블로커_triage`. **불일치 시 master 우선.**

## 3. 생성할 GA4 5객체 (원천 = events 샤드 UNION)

| SILVER 객체 | 그레인 | 핵심 원천(VARIANT FLATTEN) | 충족 GOLD |
|---|---|---|---|
| `GA4_EVENT` | 이벤트 1건(event_timestamp×user×event) | events 전 샤드 | FGA |
| `GA4_TRAFFIC_SOURCE` | utm 조합(source/medium/content/term) | `session_traffic_source_last_click` | DIM_GA_SOURCE |
| `GA4_EVENT_DIM` | 이벤트 정의(category/label/action) | `event_name`·`event_params` | DIM_GA_EVENT |

| `GA4_IDENTITY` | 식별자 1건 | `user_id`·`user_pseudo_id`+URL 파라미터 | DIM_MEMBER_IDENTITY(GA측)→S-7 |

DDL 초안 = `_archive/09_SILVER_DDL_20260702.sql`의 GA4 5테이블. 착수 시 `08_SILVER_테이블DDL_20260714.sql`(STEP 6)로 이관.

## 4. ⛔ 필수 가드레일 (이번 세션 리뷰에서 실제 터진 오류 — 반드시 예방)

1. **연·월·일자는 절대 텍스트 컬럼을 TRY_TO_NUMBER 하지 마라.**
   - AGENCY에서 텍스트 `YEAR`='2025년'/`MONTH`='03월'을 파싱하다 **96% NULL**이 발생했다.
   - GA4: `event_date`는 TEXT `'YYYYMMDD'`, `event_timestamp`는 **마이크로초 정수**다.
   - 규칙: `EVENT_DATE = TO_DATE(event_date,'YYYYMMDD')` (또는 `TO_TIMESTAMP(event_timestamp/1e6)`에서 파생). 연·월이 필요하면 **이 DATE에서 파생**(`YEAR()`/`MONTH()`). 파생 후 **NULL 0건**을 DQ로 검증.
2. **SILVER에서 월 롤업·집계 금지 → GOLD.**
   - AGENCY_COST(월 SUM 롤업)를 만들었다가 §3 위반으로 **제거**했다.
   - GA4 **비가산 지표(평균세션시간·이탈율, O1)**는 SILVER에 **원자료(raw)로만** 적재하고 율/평균 계산은 GOLD/SV에서. cohort·세션 집계도 GOLD.
3. **단방향 유지**: SILVER는 **BRONZE만 참조**. SILVER→SILVER 파생 금지(신원 브리지 S-7은 별도 교차소스 예외로 격리).
4. **멱등 적재**: `INSERT OVERWRITE` 사용. 전기간 샤드 재입고 시 재실행만으로 갱신되게.
5. **DQ 게이트 필수(생략 금지)**: 적재 직후 **① 행수 대사(SILVER=원천 샤드 UNION 합) ② PK 유일성(중복 0) ③ 파생 DATE/연월 NULL 0 ④ 조인 fan-out**. AGENCY에서 이 대사를 안 했다면 96% NULL을 놓쳤다.
6. **메타 4컬럼**: 모든 SILVER 테이블에 `DW_SOURCE_SYSTEM`('GA4')·`DW_SOURCE_TABLE`·`DW_LOAD_TS`·`DW_UPDATE_TS`(+`DW_BATCH_ID`).

## 5. GA4 특수 기술 규칙 (docs 07/12/13)

1. **컬럼은 대소문자 구분(소문자 따옴표 식별자)**. `event_params` 등 VARIANT. `SELECT *` 금지 → **명시 30컬럼**.
2. **중첩 VARIANT는 `LATERAL FLATTEN`**, leaf는 `::` 캐스팅. float 값은 `double_value`(NOT `int_value`).
   - 예: `event_params` 배열 → `LATERAL FLATTEN(input => event_params)` 후 `f.value:key::string`, `f.value:value.double_value::float`.
3. **트래픽 소스는 `session_traffic_source_last_click`**(VARIANT) 사용 = GA4 UI 일치. `traffic_source`/`collected_traffic_source` 아님.
4. **샤드 UNION**: `10_dbt_pipeline/macros/ga4_union_shards.sql`의 **대소문자 안전 수정본** 기준. 문서 07/12/13 예시 매크로는 소문자 샤드 버그 참고용. 매크로의 `SELECT *`는 **명시 30컬럼**으로 고정.
5. **LATERAL 안 VALUES절에 외부 컬럼 참조 금지**(ERP에서 `CORRELATION` 오류 발생). 언피벗/전개는 **CTE+UNION ALL** 또는 `FLATTEN` 사용.
6. **all-NULL 잡컬럼 주의**: `app_info`·`event_dimensions`·`publisher`는 NUMBER(사실상 빈 컬럼) — 매핑 대상 아님.
7. **병렬 SQL 자제**: credential cache 락 경합이 있었다 → CREATE/INSERT는 **순차 실행**.

## 6. 블로커 triage (착수 전 분류 — `11_SILVER_블로커_triage` 준수)

- **Q1(비블로커)** — `user_id`↔CRM 회원번호: 형식 일치(조인 성립)하나 **채움률 4.22%**(1,290명/287,025). descope 아님 → 커버리지 경고를 달고 진행. 회원단위 GA 지표는 부분 커버리지임을 SV에 명시. 미설정 구간은 `user_pseudo_id`+`transaction_id` 간접연계 검토.
- **O1(비블로커)** — 비가산 지표: raw 적재(가드레일 4-2).
- **O2(설계결정)** — `DIM_DEVICE` APP 정의: GA4 `platform`(WEB/IOS/ANDROID) vs 앱SDK. AGENCY 기기 차원과 **conform 규칙 확정 후** 확정(GOLD 단계와 조율).
- **하드 블로커** — 전기간 샤드 입고(§2).

## 7. 실행 순서 (ERP·AGENCY에서 검증된 저오류 루틴)

1. `08`에 GA4 5객체 **DDL append(STEP 6)** → 테이블 생성(순차, `CREATE OR REPLACE`, 멱등).
2. `09`에 GA4 **적재쿼리 append(STEP 6)** → 샤드 UNION + FLATTEN 정제 `INSERT OVERWRITE`.
3. **DQ 게이트 실행**(§4-5). 실패 시 수정→재적재 루프.
4. 통과 후 **S-7 신원 브리지** `IDENTITY_MEMBER_XREF` 신설(입력: CRM 회원번호 + GA `user_id` + 결연아동코드, `MATCH_METHOD`/`MATCH_CONFIDENCE`). 위치(SILVER vs GOLD)는 S-7에서 확정.
5. **문서 이력 기록**: `04`(GA4 §5)·`02`(master §2 실측표·§7 step2/3)에 적재 행수·DQ 결과·리뷰 이력 반영. 08/09 파일이 **실제 실행본과 일치**하는지 확인(실행 중 수정 시 파일 동기화 — AGENCY에서 파일이 실패본으로 남았던 사례 주의).
