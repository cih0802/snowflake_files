## 3. 트랙별 착수 게이트 (결론)

### 🥇 GA4 (트랙 B, 1차)
- **하드 블로커 = 1건**: 전기간 샤드 입고(외부 커넥터). **이것만이 본착수 차단 요인.**
- 나머지(Q1·O1·O2)는 🔵/🟠 → 입고 완료 즉시 착수 가능.
- **선행 가능(입고 前)**: `_archive/09` GA4 5객체 DDL을 `08`로 이관 + 1일 샤드로 FLATTEN 적재 **PoC**(멱등 INSERT OVERWRITE → 전기간 입고 시 재실행). 구조·변환 로직을 미리 검증해 critical path 단축.

### 🥈 ERP (트랙 C, 2차)
- **하드 블로커 = 0건**(BRONZE_ERP 적재 완료). GA4와 병렬 가능.
- **즉시 착수**: `ERP_BUDGET_ITEM`(✅ 매핑) + `ERP_BUDGET`(◐ 편성/집행).
- **descope(스키마-only)**: `ERP_BIZ_TARGET`(E-6) — CREATE TABLE만, 적재 보류(현업 원천 대기).
- **grain 제약 명시 후 진행**: Q10 → 부서 grain까지. 캠페인 축·ROI는 현업 연결키 확보 시.
- **부분 대기**: FBD 모금성비용/광고비는 AGENCY `AGENCY_COST` 결합(3차 이후).

### 🥉 AGENCY (트랙 D, 3차)
- **하드 블로커 = 0건**(BRONZE_AGENCY 3테이블 적재 완료).
- **설계결정 블로커 = DDL 前 확정**(모두 내부 해소, 외부 의존 없음): ① `_SOURCE_SYSTEM` 부여규칙(A-2/Q9) ② 인입콜 `TRY_TO_NUMBER` ③ 명/건·CVR measure 정의 ④ measure 불균일 UNION 패딩 ⑤ 캠페인 이름매칭 크로스워크 ⑥ DGT 파생 원천값vs재계산.
- 확정 후 착수. `FAD` 전환분=GA4(1차) 완료 후 결합, `FBD` 비용=ERP(2차) 결합 → **선행 트랙 후 착수가 효율적**.
