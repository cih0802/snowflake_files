## 3. 트랙 C 선결·실측 검토 (S-6 착수 전 게이트)

> **[실측 2026-07-13]** `BRONZE_ERP.BDGT_ACMSLT_LEDGER` = 1테이블·62컬럼·2,041행. 예산·집행 원장(예산과목 장/관/항/목/세목/세세목 + 예산단위명 + 재원 + 월별 편성/추경/조정/집행).

- **`DIM_BUDGET_ITEM`(세세목·예산구분)** → ✅ 매핑 가능.
- **`FBD`(편성/집행)** → ◐ 편성·집행 measure는 있으나 **모금성비용·광고비는 원장에 없음** → AGENCY로 보강 검토.
- **`FACT_TARGET_BIZ`(사업목표)** → ❌ **원장은 사업목표가 아님 → FTG-B 원천 여전히 부재**. 현업 별도 원천(사업계획 #152~155) 필요.
- **Q10 — 세세목↔캠페인 매핑(O3)**: 캠페인/조직 연결키 없음(조직=이름, 코드X) → **캠페인별 예산 ROI 불가**. 부서 grain까지만 가능(현업 확인 — 결론7/O3·E-1·E-4).

> **⛔ 우리끼리 불가 (데이터 부재 → 스키마-only)**: `FACT_TARGET_BIZ`(사업목표) · `FBD` 모금성비용 원천. → 입고/현업 확인 대기.

## 4. ERP 정제 규칙

- **공통 규칙**: master §3 (타입·NULL·코드→라벨·메타 4컬럼·금액 원단위) 준수.
- **ERP 특수**:
  - 예산과목 6단계(장/관/항/목/세목/세세목)는 계층 컬럼 보존 → `DIM_BUDGET_ITEM` 계층.
  - 월별 편성/추경/조정/집행은 **wide→long 정규화 검토**(FBD 월 grain 적재 시).
  - 조직=이름(코드 없음) → `DIM_ORG`(CRM) conform 시 **이름 매칭 크로스워크** 필요·미매칭 플래그.
  - 금액 원단위 보존.
