# GN_DW 데이터 활용 가이드 (표준 지표·도메인 관계)

> GN_DW 데이터 웨어하우스의 **현업 활용 가이드** — 기술 개념도(`03_GN_DW_개념도.html`)와 함께 활용하는 현업(CRM·마케팅·재무) 대상 단일 자료.
> 관점: "어떤 업무 의사결정을 지원하는가?" · "데이터를 신뢰할 수 있는가?"
> 본 자료의 지표·정의는 **실제 `GN_DW.GOLD` 32객체 구성**에 근거함(2026-07-16 실측 갱신 — DIM_MEMBER SCD2 활성 반영).

## 1. 비즈니스 질문 → 마트 매핑표 (Use-Case Map)

> 테이블명이 아닌 **업무 질문**을 기준으로 데이터 마트를 연결합니다.

| 현업의 비즈니스 질문 | 조회할 최종 데이터 (GOLD) | 상태 | 해결되는 업무 리스크 |
|---|---|---|---|
| "지난달 캠페인·후원사업별 신규개발/이탈(감액+취소) 실적은?" | `WIDE_MEMBER_MONTHLY` | ✅ 사용가능 | 부서별 엑셀 수계산 실적 불일치 해소 |
| "회원 상태·지역·연령대별 활동회원·납입회비 추이는?" | `WIDE_MEMBER_MONTHLY` | ✅ 사용가능 | 회원 세그먼트 분석 표준화 |
| "특정 알림톡/서신 발송 후 +5일 내 증액·서신참여한 회원은?" | `WIDE_SERVICE_EVENT` | ✅ 사용가능 | 발송↔반응 데이터 자동 결합 |
| "행사별 참여 회원과 후원 연계 실적은?" | `WIDE_EVENT_PARTICIPATION` | ✅ 사용가능 | 행사 ROI 추적 |
| "조직·부서별 회원개발 목표 대비 달성률은?" | `WIDE_TARGET_DEV` | ✅ 사용가능 | 목표관리 표준화 |
| "예산 편성 대비 집행 현황은? (조직·예산과목별)" | `WIDE_BUDGET` | ◐ 부분 | 편성/집행 O · 모금성비용/광고비는 원천대기(E-1) |
| "디지털 광고 매체비·노출·클릭 성과는?" | `WIDE_AD_PERFORMANCE` | ◐ 부분 | 매체 성과 O · **캠페인 연결·ROAS는 대기(Q10/O5)** |
| "채널 광고 대비 CRM 후원 전환율(ROAS)은?" | `WIDE_AD_PERFORMANCE` + 신원브리지 | 🔜 로드맵 | 신원브리지 활성(1일 기반)·광고 연결키(Q10/O5) 대기 |
| "연/추경 사업목표 대비 달성률은?" | `WIDE_TARGET_BIZ` | ⛔ 대기 | ERP 사업목표 원천 미입고(E-6)·`FACT_TARGET_BIZ` 0행. 뷰·구조는 준비완료(입고 시 자동 표출) |

**범례**: ✅ 즉시 사용가능 · ◐ 부분 사용가능(일부 컬럼 대기) · 🔜 로드맵 · ⛔ 원천 입고 대기

## 2. 핵심 지표 표준화 정의서 (Standard KPI Glossary)

> Silver·Gold 계층의 목적 = **부서 간 지표 표준화(Single Source of Truth)**.
> 아래 산출 기준은 `WIDE_MEMBER_MONTHLY`·`WIDE_SERVICE_EVENT` 컬럼 정의(정본) 근거.

| 표준 지표 | 정의 / 산출 기준 | 저장 위치 |
|---|---|---|
| **개발(명) `DEV_MEMBERS`** | 당월 신규 개발 회원 수 (COUNT) | `WIDE_MEMBER_MONTHLY` |
| **개발(건) `DEV_CNT`** | 당월 개발 금액 기준 건수 = SUM(개발금액)/10,000 | `WIDE_MEMBER_MONTHLY` |
| **활동회원(명) `ACTIVE_MEMBERS`** | 월말 기준 정상 납입 이력 or 유효 결연 상태 회원 수 | `WIDE_MEMBER_MONTHLY` |
| **월말활동회원 `MONTH_END_ACTIVE_CNT`** | 월말 시점 활동 상태 건수 (전월말 대비 증감 추적) | `WIDE_MEMBER_MONTHLY` |
| **이탈(건) `CHURN_CNT`** | **후원 취소 + 감액 금액 합산** = SUM(취소+감액)/10,000 *(신규 표준지표)* | `WIDE_MEMBER_MONTHLY` |
| **감액(건) `DECREASE_CNT`** | SUM(감액금액)/10,000 | `WIDE_MEMBER_MONTHLY` |
| **증액(명/건) `INCREASE_MEMBERS`/`_CNT`** | 당월 증액 회원 수 / 건수 | `WIDE_MEMBER_MONTHLY` |
| **미납(건) `UNPAID_CNT`** | 당월 미납 발생 건수 (캠페인별·상태별 분해 컬럼 별도 제공) | `WIDE_MEMBER_MONTHLY` |
| **납입회비(원) `PAID_FEE`** | 당월 실납입 회비 총액 (원 단위) | `WIDE_MEMBER_MONTHLY` |
| **회원구분 `MEMBER_TYPE`** | 개인 / 기업 / 단체 (MM018 코드 기준) | `DIM_MEMBER` |
| **회원상태 `MEMBER_STATUS`** | 활동·미납·중단 등 상태(MM010) — **SCD2 시점이력**(EFFECTIVE_FROM/TO·IS_CURRENT) | `DIM_MEMBER` |
| **서신참여(명/건) `LETTER_PART_*`** | 발송 후 서신 참여 회원/건수 | `WIDE_SERVICE_EVENT` |
| **+5일차 반응 `D5_*`** | 발송일 기준 +5일 내 증액·서신·선물금·중단 반응 | `WIDE_SERVICE_EVENT` |

> **회원 통합 표준화**: 정기(FDRM)/일시(ONCE)는 원천 CRM에서 별도 테이블 → Silver `CRM_MEMBER`에서 단일 회원으로 통합(회원번호 `S` 접두로 판별). ※ `DIM_MEMBER.MEMBER_TYPE`은 **개인/기업/단체(MM018)**를 담으며, 정기/일시 구분을 지표로 노출하려면 별도 파생 필요(엔지니어 확인 항목).

## 3. Before & After (도입 효과)

| 구분 | Before | After |
|---|---|---|
| 실적 마감 | 부서 간 수치 불일치 → 원인 규명 다수 인일 소요 | Gold 표준 지표로 단일 진실 공급원(SSOT) 확보 |
| 광고·CRM 연계 | 채널 성과↔CRM 결제 수작업 매핑 | Silver ID 통합 → WIDE 뷰에서 자동 결합 |
| 지표 정의 | 부서별 편차("활동회원" 수치 회의마다 상이) | KPI Glossary 고정 산식 + BI 직결 |

### Before (현행 업무의 제약)
- 실적 마감 시 **마케팅·모금·재무팀 간 수치 불일치** → 원인 규명에 다수 인일(人日) 소요
- 채널별 광고 성과와 CRM 실결제 연계에 **수작업 매핑** 의존
- 지표 정의의 부서별 편차로 "활동회원"·"이탈" 수치가 회의마다 상이

### After (GN_DW 도입 후)
- **Silver**: 회원번호 ID 통합·코드 표준화 → 부서 공통 기준 확보
- **Gold**: 표준 지표 사전 계산 (`FACT_*` 3,700만~3,800만 행 규모 집계 완료)
- **WIDE 뷰**: BI 툴(Tableau·Power BI)에서 부서 간 일치된 표준 지표 보고서를 즉시 조회

### 향후 로드맵 (원천 입고 대기)
| 항목 | 현재 | 활성 조건 |
|---|---|---|
| GA4 트래픽 ↔ CRM 후원 결합(ROAS) | 🔜 신원브리지 `DIM_MEMBER_IDENTITY` 활성(1,274명·1일 기반) | 전기간 입고(G-5)+광고 연결키 |
| 광고 캠페인별 ROI | 🔜 캠페인 연결키 미해소 | 캠페인 매칭키 회신(Q10/O5) |
| 사업목표 달성률 | ⛔ `WIDE_TARGET_BIZ`·`FACT_TARGET_BIZ` 구조준비·0행 | ERP 사업목표 원천 입고(E-6) + 단위 확정(금액/건) |
| 예산 내 모금성비용·광고비 | ◐ NULL | AGENCY 비용 보강(E-1) |

## 4. 비즈니스 도메인 관계도 (Domain Model)

> 시스템 키·이력관리 방식 등 구현 요소를 배제하고, 업무 언어로 표현한 전체 관계 조망.

```
                         ┌─────────────────┐
              ──가입──▶   │  캠페인 / 브랜드 │  ──귀속──▶  ┌────────────┐
              │          └─────────────────┘             │ 조직 / 부서 │
   ┌──────────────┐                                       └────────────┘
   │ 회원 / 후원자 │ ──참여──▶  [ 행사 · 서신 · 발송(알림톡) ]
   └──────────────┘
              │
              ──결제──▶  [ 납입 · 청구 · 예산(편성/집행) ]
```

- **중심 축**: `회원/후원자` ──가입──▶ `캠페인/브랜드` ──귀속──▶ `조직/부서`
- **활동 축**: `회원/후원자` ──참여──▶ `행사/서신/발송`
- **재무 축**: `회원/후원자` ──결제──▶ `납입/청구/예산`

> 각 개념은 기술 개념도의 `DIM_MEMBER`·`DIM_CAMPAIGN`·`DIM_ORG`·`DIM_EVENT`·`DIM_SERVICE`·`FACT_BUDGET`에 매핑되며, 현업 사용 시 별도 인지가 필요 없습니다.
