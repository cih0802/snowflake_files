# GN_DW paid 계정 재현(replay) 런북

> **목적:** trial에서 검증(Phase-1)한 GN_DW를 **신규(paid) 계정에 처음부터 재현**하는 의존성-안전 실행 순서.
> **핵심 원칙:** 모든 DDL은 `GN_DW_ADMIN`으로 생성 → 객체가 처음부터 ADMIN 소유. dbt 적재는 `GN_DW_ENGINEER`(최소권한). WIDE 뷰는 dbt(ENGINEER)가 생성 → ENGINEER 소유.
> **⚠️ `07_ENVIRONMENT_RBAC_setup.sql`은 통째로 실행하지 말 것** — 내부 섹션이 객체 존재를 전제하므로 아래처럼 **분할 실행**한다.

---

## 왜 파일 순서 그대로 실행하면 안 되나 (요약)

| 문제 | 원인 | 해결 |
|---|---|---|
| 07 통째 실행 실패 | `§B.5`·`§D`·`§G`가 스키마/테이블/GOLD 컬럼 존재를 전제 | 07을 **§A~C / §D·§D.5 / §E·§G / §F**로 분할 |
| SILVER/GOLD DDL CREATE SCHEMA 실패 | 두 DDL이 `USE ROLE GN_DW_ADMIN`인데 ADMIN에 CREATE SCHEMA 없음 | **Phase 1에서 DB 소유권을 ADMIN에 부여** |
| DBT PROJECT ADD VERSION 실패 | 신규 계정엔 프로젝트가 없음 | **`CREATE DBT PROJECT`**(최초) 사용 |
| OPS 스키마 부재 | 어느 DDL도 OPS를 만들지 않음 | Phase 1에서 **`CREATE SCHEMA GN_DW.OPS`** |
| SV DDL 실패 | SV가 `SERVING.DIM_MONTH`/`DIM_MEMBER_CURRENT` 참조 | **§E+§G(SERVING+helper 뷰)를 SV DDL 전에** |

> 참고: 신규 계정은 모든 객체를 ADMIN으로 생성하므로 `07 §B.5`(소유권 이관)와 WIDE 뷰 이관은 **불필요**(no-op). 이 두 단계는 기존 trial 계정(ACCOUNTADMIN 소유로 시작)에서만 필요했다.

## 실행 순서

### Phase 0 — 기반 (07 §A~C)
`07_ENVIRONMENT_RBAC_setup.sql`의 **A(웨어하우스)·B(역할6+계층)·C(WH grant) 섹션만** 실행. `§B.5` 주석 이후는 **실행 금지**.
- 산출: WH 3(ETL/ANALYTICS/DEV) · 역할 6 · 역할 계층 · WH USAGE grant.

### Phase 1 — DB · ADMIN 소유 · OPS 스키마
```sql
USE ROLE SYSADMIN;
CREATE DATABASE IF NOT EXISTS GN_DW COMMENT='굿네이버스 데이터웨어하우스';

USE ROLE ACCOUNTADMIN;
-- DB 소유권을 ADMIN에 → 이후 모든 스키마/테이블이 ADMIN 소유로 생성됨
GRANT OWNERSHIP ON DATABASE GN_DW TO ROLE GN_DW_ADMIN COPY CURRENT GRANTS;

USE ROLE GN_DW_ADMIN;
CREATE SCHEMA IF NOT EXISTS GN_DW.OPS
  COMMENT='ETL 운영 인프라 전용 (DBT PROJECT 등)';
```

### Phase 2 — BRONZE 구조 + 적재
1. **세션 role을 `GN_DW_ADMIN`으로** 두고 `50_handoff/02_데이터마이그레이션 GN_DW_BRONZE_DDL_20260713.sql` 실행.
   - `create database if not exists GN_DW`는 no-op, BRONZE 4스키마 + 48테이블이 ADMIN 소유로 생성.
   - ⚠️ 이식성 위해 이 파일엔 role 프리앰블이 없다 → 반드시 `USE ROLE GN_DW_ADMIN;`을 먼저 실행.
2. **BRONZE 데이터 적재** (paid는 전기간 전수). 원천→stage→`COPY INTO` 또는 마이그레이션 절차(`50_handoff` / `setup_20260702.sql` 패턴). GA4는 VARIANT라 Parquet 경로.

### Phase 3 — SILVER / GOLD 구조 (빈 테이블)
3. `04_silver_design/08_SILVER_테이블DDL_20260714.sql` 실행 (파일 내 `USE ROLE GN_DW_ADMIN` 프리앰블 有).
4. `03_top-down_gold/06_DDL.sql` 실행 (동일). → SILVER 32 + GOLD 24(DIM15+FACT9) 구조·컬럼 COMMENT·FK 생성.

### Phase 4 — 적재 권한 + dbt 배포·실행
5. `07`의 **`§D`(스키마 소비 grant) + `§D.5`(ENGINEER 적재 grant)** 실행 (`USE ROLE GN_DW_ADMIN`).
6. DBT PROJECT **최초 생성**(ALTER 아님) + 실행:
```sql
USE ROLE GN_DW_ADMIN; USE WAREHOUSE GN_DW_DEV_WH;
CREATE DBT PROJECT GN_DW.OPS.DW_PIPELINE
  FROM 'snow://workspace/USER$.PUBLIC."snowflake_files"/versions/live/10_dbt_pipeline';
GRANT USAGE, MONITOR ON DBT PROJECT GN_DW.OPS.DW_PIPELINE TO ROLE GN_DW_ENGINEER;

-- 적재 실행(ENGINEER = 최소권한, CREATE TABLE 없이 적재)
USE ROLE GN_DW_ENGINEER; USE WAREHOUSE GN_DW_ETL_WH;
EXECUTE DBT PROJECT GN_DW.OPS.DW_PIPELINE ARGS='build';
```
- 기대: `56 incremental models + 9 view models + 167 tests`, `ERROR=0`. WIDE 뷰 9는 ENGINEER 소유로 생성.
- `profiles.yml`은 이미 role=`GN_DW_ENGINEER`, wh=`GN_DW_ETL_WH` → 별도 수정 불필요.
- 같은 워크스페이스면 dbt 파일 복사 불필요(경로 그대로 참조).

### Phase 5 — SERVING SV + Agent
7. `07`의 **`§E`(SERVING 스키마) + `§G`(helper 뷰 `DIM_MONTH`·`DIM_MEMBER_CURRENT`)** 실행 (`USE ROLE GN_DW_ADMIN`). ← **SV DDL보다 반드시 먼저** (SV가 두 뷰를 참조).
8. `05_SV-Agent_ai/05_SV_DDL.sql` 실행 (`USE ROLE GN_DW_ADMIN`) → SV 5.
9. `05_SV-Agent_ai/09_AGENT_spec_구현.sql`의 **[1-ALT] `CREATE AGENT ... FROM SPECIFICATION`** 블록을 `GN_DW_ADMIN`으로 실행 → Agent 2(AGENT_MEMBER·AGENT_OVERALL). owner=ADMIN.

### Phase 6 — CoWork 연결 + 테스트
10. `07`의 **`§F`(CoWork object `SNOWFLAKE_INTELLIGENCE_OBJECT_DEFAULT`)** 실행 (`USE ROLE ACCOUNTADMIN`) + SV/Agent별 `USAGE`를 소비 3역할(ANALYST/VIEWER/SERVICE)에 부여 + SI에 `ADD AGENT`(09 [4]).
11. `05_SV-Agent_ai/12_paid_테스트_실행가이드.md`대로 NL 스모크 테스트. paid 계정은 `DATA_AGENT_RUN` 가능(trial 차단 해제).

## 소유·권한 최종 상태 (설계 정합)
- **소유 = `GN_DW_ADMIN`**: DB · 전 스키마 · 전 테이블 · SV · Agent · DBT PROJECT (구조·COMMENT 보호).
- **적재 = `GN_DW_ENGINEER`**: GOLD `INSERT/UPDATE/DELETE/TRUNCATE` · SILVER `INSERT/TRUNCATE` · WIDE 뷰 소유 · DBT PROJECT `USAGE`. **GOLD `CREATE TABLE` 없음**.
- **소비 = ANALYST/VIEWER/SERVICE**: GOLD/WIDE `SELECT` + SV/Agent `USAGE`.
- **계정 레벨 = ACCOUNTADMIN 유지**: 네트워크/인증 정책 · Resource Monitor · CoWork object · `SNOWFLAKE.CORTEX_*`.
