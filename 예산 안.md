# Snowflake 웨어하우스별 예산 설정 가이드

## 예산 설정 방법

1. **Resource Monitor** — 일간/주간/월간 단위로 크레딧 쿼타 설정, 임계값 도달 시 알림 또는 자동 중단
2. **Custom Budget** — 월 단위 크레딧 한도, 초과 예상 시 알림 (중단은 안 됨)

## 웨어하우스 분리 시 권장 예산 비율

총 월 크레딧 예산이 있다고 가정할 때:

| 웨어하우스 | 용도 | 월 비율 | 주간 리셋 권장 |
|---|---|---|---|
| `WH_PIPELINE_LIGHT` | 가벼운 ELT 파이프라인 | 15~20% | 주간 모니터링 |
| `WH_PIPELINE_HEAVY` | 무거운 배치 파이프라인 | 35~40% | 주간 모니터링 |
| `WH_ANALYTICS` | 데이터 분석/조회 | 25~30% | 일간 모니터링 |
| `WH_AI_CHATBOT` | AI 챗봇/Cortex 연결 | 15~20% | 일간 모니터링 |

## 설정 예시 (월 1000 크레딧 기준)

```sql
-- 가벼운 파이프라인용 (월 200 크레딧, 주간 리셋)
CREATE RESOURCE MONITOR rm_pipeline_light WITH CREDIT_QUOTA = 50
  FREQUENCY = WEEKLY START_TIMESTAMP = IMMEDIATELY
  TRIGGERS ON 80 PERCENT DO NOTIFY
           ON 100 PERCENT DO SUSPEND;
ALTER WAREHOUSE WH_PIPELINE_LIGHT SET RESOURCE_MONITOR = rm_pipeline_light;

-- 무거운 파이프라인용 (월 400 크레딧, 주간 리셋)
CREATE RESOURCE MONITOR rm_pipeline_heavy WITH CREDIT_QUOTA = 100
  FREQUENCY = WEEKLY START_TIMESTAMP = IMMEDIATELY
  TRIGGERS ON 80 PERCENT DO NOTIFY
           ON 100 PERCENT DO SUSPEND;
ALTER WAREHOUSE WH_PIPELINE_HEAVY SET RESOURCE_MONITOR = rm_pipeline_heavy;

-- 분석 조회용 (일간 15 크레딧)
CREATE RESOURCE MONITOR rm_analytics WITH CREDIT_QUOTA = 15
  FREQUENCY = DAILY START_TIMESTAMP = IMMEDIATELY
  TRIGGERS ON 80 PERCENT DO NOTIFY
           ON 100 PERCENT DO SUSPEND;
ALTER WAREHOUSE WH_ANALYTICS SET RESOURCE_MONITOR = rm_analytics;

-- AI 챗봇용 (일간 7 크레딧)
CREATE RESOURCE MONITOR rm_ai_chatbot WITH CREDIT_QUOTA = 7
  FREQUENCY = DAILY START_TIMESTAMP = IMMEDIATELY
  TRIGGERS ON 90 PERCENT DO NOTIFY
           ON 100 PERCENT DO SUSPEND_IMMEDIATE;
ALTER WAREHOUSE WH_AI_CHATBOT SET RESOURCE_MONITOR = rm_ai_chatbot;
```

## 핵심 팁

- **파이프라인**은 주간 단위가 적합 (배치 스케줄에 유연성 부여)
- **분석/AI**는 일간 단위가 적합 (폭주 쿼리 조기 차단)
- AI 챗봇은 `SUSPEND_IMMEDIATE` 권장 (예측 불가한 사용 패턴)
- 50%, 75%, 90% 등 다단계 `NOTIFY` 트리거로 조기 경고 추가 추천
