# 2단계 — 7 SV 구조 설계

> 각 SV = **1 base FACT(grain) + conformed DIM(star)**. relationship 조인키는 **배포 실물 컬럼**만 사용(추정 금지). measure는 가산성 분류(F/S/N)와 `default_aggregation` 명시(R5). derived 비율/누계/증감은 물리 비적재 → 3단계 metric expression(본 문서는 base measure·차원까지만).

---

## ⚠ 정정 로그 (2026-07-22 — A1/A3 적재 + 실측 재검증; 후속 3단계 오류 방지)

> **§0.3·§0.6·§1~§5의 적재/행수 진술은 2026-07-21 A1/A3 적재 *이전* 스냅샷이다. 충돌 시 아래 정정이 우선하며, 실배포 활성 스코프의 정본은 `05_SV_DDL.sql`(재실측 COUNT_IF 기준)이다.** 설계 구조(조인키·fan-out·가산성)는 유효.
>
> 1. **FMM 행수**: 37.8M/37,792,336 → **40,054,883**(실측). 37,792,336은 `HAS_BILLING=TRUE` 부분집합(구 스파인)일 뿐 전체 행수 아님. grain (MONTH_KEY, MEMBER_DK)는 **여전히 유일**(40,054,883 = distinct) → PK·fan-out 안전.
> 2. **A1 적재 → §0.6 "FMM DEV/STOP 전건 0" 무효**: 실측 `DEV_CNT`=2,970,417 · `STOP_CNT`=972,376 적재됨 → 개발/중단 총건 **활성**(SV_MEMBER_MONTHLY). 단 UNPAID_CNT·ACTIVE_CNT·INCREASE 등 잔여 카운트·NEW_EXISTING_FLAG·FK(CAMPAIGN/PAYMENT/SPONSORSHIP/REASON_SK)는 **여전히 0**(비활성 유지).
> 3. **A3 적재 → §0.6 "FSE SERVICE_SK=0" 무효**: `SERVICE_SK`=99.97%·`SEND_TITLE`=36.6M 적재됨 → **서비스구분(DIM_SERVICE: SUBTYPE·CHANNEL) 분해 활성**(SV_SERVICE). SUCCESS/FAIL/OPEN/D5·CAMPAIGN_SK는 여전히 0(비활성 유지).
> 4. **FME grain 정정(중요)**: §2의 "grain=DATE_SK×MEMBER_DK×EVENT_TYPE"는 **유일하지 않음** — 실측 4,633,105행 vs 해당 3키 distinct 4,052,797(약 12.5% 중복). 진짜 grain은 더 세밀(추가 FK / 동일일·동일유형 복수사건). SV 집계는 무해(FME는 조인 다(多)측·dim 일(一)측 → fan-out 없음)하나 **PRIMARY KEY 선언 근거는 아님** → 3단계 DDL의 FME PK는 제거/실grain 반영 예정(SQL 정정은 별도 세션).
> 5. **용어 정합(`default_aggregation`)**: 이는 **Cortex Analyst YAML 모델** 개념이며 **DDL `CREATE SEMANTIC VIEW`에는 없는 절**이다. DDL에서는 **METRIC=집계식(SUM/COUNT)·FACT=행수준식**으로 가산성을 표현한다. 본문의 "F → default_aggregation: sum"은 DDL에서 **"F → SUM METRIC 정의"** 로 읽는다(실배포 `05_SV_DDL.sql`=DDL 방식).
> 6. **S-class(준가산) 강건화**: 시점재고(ACTIVE_CNT 등) 다월 합산 방지는 instruction보다 **DDL `NON ADDITIVE BY <시간 dim>`**(semi-additive metric) 구문이 구조적으로 강건 → Phase-2 활성 시 채택 권장. (현재 S measure 전건 미적재라 유예.)
> 7. **전체설계 §1~§5 표 주의**: FK·measure 미적재분까지 **전량 열거**(설계 완전성 목적)한다 → 실제 노출 대상이 아니다. **실배포 스코프 = §0.6(정정본) ∩ `05_SV_DDL.sql`**. §1~§5의 dimension/measure는 적재 완결 시 활성화하는 청사진으로만 사용.

## 0. 공통 설계 규칙 & 필수 conformed helper 뷰

### 0.1 fan-out 차단 (원칙10·R1) — **선행 필수 뷰 2종**
배포 실물 확인 결과 raw 조인 시 두 곳에서 fan-out 발생 → SERVING에 conformed helper 뷰를 만들어 SV가 이를 참조한다.

| 위험 | 원인(실측) | 차단 helper 뷰 (GN_DW.SERVING) | PK |
|---|---|---|---|
| **월×일 fan-out** | 월팩트 `MONTH_KEY` → `DIM_DATE`(일 grain 16,437행) 조인 시 월당 ~28~31× 증폭 | `DIM_MONTH` = `SELECT DISTINCT MONTH_KEY, YEAR, MONTH, QUARTER FROM GOLD.DIM_DATE` | MONTH_KEY (unique) |
| **SCD2 회원 fan-out** | `DIM_MEMBER` SCD2 7,925,716행(회원당 평균 4.5·최대 218버전) → `MEMBER_DK` 조인 시 버전수만큼 증폭 | `DIM_MEMBER_CURRENT` = `SELECT … FROM GOLD.DIM_MEMBER WHERE IS_CURRENT = TRUE` | MEMBER_DK (현재행 유일=1,763,065) |

- 월팩트(FMM·FBD·FTG_D·FTG_B)는 **DIM_DATE 직접 조인 금지 → DIM_MONTH 사용**.
- 회원 차원 속성(성별·지역·연령대·회원상태 등)은 **DIM_MEMBER_CURRENT** 참조(현재 스냅샷). as-was 이력 분석 요구 없음(DEC-2 유사).
- 검증(3단계 DoD): `SELECT FROM SEMANTIC_VIEW(...)` measure 합 = 단일 FACT `SELECT SUM(...)` 합 일치(fan-out 스모크).

### 0.2 조인키 타입 주의 (O12)
- `MEMBER_DK` = **VARCHAR(10)** (정규 7자리 leading-zero / 일시 `S`+8자리). **NUMBER 캐스팅 절대 금지**. 조인은 문자열 그대로.
- SK(surrogate) 계열은 NUMBER. `MONTH_KEY`=NUMBER(YYYYMM), `DATE_SK`=NUMBER(YYYYMMDD).

### 0.3 가산성 분류(R5) — measure `default_aggregation` 규칙
| 클래스 | 의미 | 예 | default_aggregation | 시간 합산 |
|---|---|---|---|---|
| **F**(flow, 완전가산) | 기간 내 발생량 | DEV_CNT·STOP_CNT·PAID_FEE·발송수 | `sum` | 월 합산 OK |
| **S**(stock, 준가산) | 특정 시점 재고 | ACTIVE_CNT·MONTH_END_ACTIVE_CNT·ACTIVE_CUM_CNT | `sum`(차원 간) + **instruction: 여러 월 합산 금지·최신월/시점값 사용** | 시간 합산 ✗ |
| **N**(비가산) | 사전집계 비율/평균 | AVG_SESSION_DURATION·BOUNCE_RATE | 직접 노출, `sum`/`avg` 재집계 금지 | ✗ |
| **D**(distinct 회원수) | 기간 내 고유 회원 | ACTIVE_MEMBERS·DEV_MEMBERS·SEND_MEMBERS | 단일 기간=`sum` OK / **다기간=`COUNT(DISTINCT MEMBER_DK)` metric**(사전집계 컬럼 SUM 시 월간 중복) | 시간 합산 ✗(중복) |

> **D 클래스 근거(실측)**: FMM grain = (MONTH_KEY, MEMBER_DK) 유일(40,054,883=distinct, 회원당 월 1행 — 2026-07-22 정정; 구 37,792,336은 HAS_BILLING 부분집합). ∴ `*_MEMBERS`는 한 달 내 차원 합산은 중복 없으나, 여러 달 합산 시 동일 회원 중복 → 기간 distinct 회원수는 `COUNT(DISTINCT MEMBER_DK)`(+필터) metric으로 정의. 사전집계 `*_MEMBERS` 컬럼의 다월 SUM 금지.

### 0.4 공통 dimension·instruction
- **시간(원칙8·R7)**: 07_메타 enum을 SV custom instruction에 명문화("조회 구간이 해당 지표 가용시점 이전이면 제외/NULL"). SV별 §에 표기.
- **NULL/Unknown**: SK=0(Unknown) 라우팅 행 존재 → dimension `sample_values`에 'Unknown' 포함, 커버리지 고지.
- **synonyms 한글 필수**(원칙6): 아래 각 dimension에 대표 한글 동의어 명시(3단계 DDL에서 확장).

### 0.5 아키텍처 비판적 검토 & 개선 (실측 반영, 2026-07-21)
> FMM 실측(grain=(MONTH_KEY,MEMBER_DK) 유일·시점 스냅샷 컬럼 자체 보유) 후 초안의 3개 결함을 교정.

| # | 결함 | 개선 | 영향 |
|---|---|---|---|

| B | **distinct 회원수 가산성** — `*_MEMBERS`를 F로 분류(다월 SUM 시 회원 중복) | **D 클래스**: 다기간 distinct는 `COUNT(DISTINCT MEMBER_DK)` metric | §0.3·각 measure표 |
| C | **목표대비 grain 충돌** — FMM엔 ORG_SK 없음(실측)·목표 FTG_D는 ORG grain → 회원 SV에 raw 얹으면 fan-out | 브리지를 FMM과 **관계 없는 독립 논리테이블(grain island)** 로 노출(§8.1). 대안: 별도 SV 분리 | §8.1 |

**시점-가변(FACT 스냅샷 사용) vs 불변(DIM_MEMBER_CURRENT) 속성 구분:**
- FACT 스냅샷(as-of 정합): 신규기존(`FMM.NEW_EXISTING_FLAG`)·개발구분·캠페인·후원사업·납입방식·사유·금액대·기간대·미납플래그(BOM/EOM).
- DIM_MEMBER_CURRENT(불변만): 성별·최초가입일·최초캠페인·가입경로·회원구분.
- ⚠ **한계 명시**: 지역·연령대는 DIM_MEMBER에만 존재(FACT 미보유) → 현재값 슬라이스만 가능. instruction에 "지역/연령대는 현재 기준" 고지.

## 8. 브리지 뷰 초안 (R1 — conformed grain 사전집계, raw 다중 FACT 조인 대체)

> 전부 GN_DW.SERVING 뷰(GOLD cross-schema 참조). SV는 이 뷰를 **단일 논리테이블**로 노출(원칙10). 3단계에서 DDL 확정·fan-out 스모크.

### 8.1 목표대비 브리지 `BRG_DEV_VS_TARGET` — MONTHLY용 (공1~3) Phase 1
- grain: MONTH_KEY × ORG_SK × DEV_TYPE
- 실적: `FME`에서 개발건 집계(개발건 ORG축은 FME, 04-61) → SUM(DEV_CNT) GROUP BY MONTH(사건일→MONTH_KEY), ORG_SK, DEV_TYPE
- 목표: `FTG_DEV`(GOAL_CNT) 동일 grain (이미 MONTH_KEY×ORG×DEV_TYPE, 7,272행 ✅)
- 조인: 실적 ⟗ 목표 ON (MONTH_KEY, ORG_SK, DEV_TYPE) → 컬럼 DEV_CNT, GOAL_CNT
- ⚠ FME 사건일 DATE_SK(YYYYMMDD)→MONTH_KEY 변환(=FLOOR(DATE_SK/100)) 후 집계.
- **⚠개선 C — grain island**: FMM엔 ORG_SK 없음(실측) → 본 브리지(ORG grain)를 SV_MEMBER_MONTHLY에 **FMM과 관계 없는 독립 논리테이블**로 노출(relationship 선언 금지). 목표대비 질의는 이 테이블 단독 라우팅. DIM_MONTH·DIM_ORG만 conform. **대안(더 순수)**: 별도 `SV_DEV_TARGET`(ORG×MONTH×DEV_TYPE)로 분리 — 계획의 "폴딩" 결정 재고 대상(사용자 판단).

### 8.2 서비스 코호트 브리지 `BRG_SERVICE_COHORT` — SERVICE용 (신35~53 유지/납입) Phase 1
- grain: 발송코호트 키 = MEMBER_DK × 발송일(DATE_SK) × SERVICE_SK × CAMPAIGN_SK
- 모집단: FSE의 발송/참여/증액/서신/선물금 회원(D5_* 포함)
- 결과상태: FME(유지/중단) · FMM(납입) 를 **회원×기준일+Nmonth** 로 사전 조인집계 → 유지회원수·납입회원수 컬럼
- 출력: 모집단수 + 코호트 결과수(유지/납입/중단) → SV에서 비율 metric
- ⚠ +5일차(D5_*)는 FSE 기적재 measure 사용, N개월 유지는 FME cohort 계산.

### 8.3 cross-source IDENTITY 브리지 `BRG_GA_MEMBER` — 공81·신32·신33 Phase 2

### 8.4 예산-실적 ROI 브리지 `BRG_BUDGET_ROI` — BUDGET용 (신9~11) Phase 2
