# 6단계 — CoWork(Snowflake Intelligence) 연결·검증

> 사용자가 배포(save/publish)한 2 Agent를 **CoWork(SI object)에 연결**하고, `07_평가셋_eval.md` ground truth로 **자연어 스모크·회귀**를 수행한다.
> **실행 주체**: 배포/USAGE grant = `GN_DW_ADMIN`(08 §4). SI `ADD AGENT` = **SI object owner(ACCOUNTADMIN)** 또는 SI object MODIFY 권한 보유 역할. 스모크 검증 = `cortex_agent_query`(읽기전용) 또는 CoWork 챗.
> **선행 상태(2026-07-22 실측)**: `SNOWFLAKE_INTELLIGENCE_OBJECT_DEFAULT` 존재 ✅ / `GN_DW.SERVING`에 Agent 0개(미배포) → **§1 선행 배포 필수**.
>
> ✅ **실행 완료(2026-07-22, CoCo 대행)**: §1 배포(save)·소유권 GN_DW_ADMIN 이전·USAGE grant(ANALYST/VIEWER/SERVICE) + §2 CoWork `ADD AGENT` 모두 완료. `SHOW AGENTS IN SCHEMA GN_DW.SERVING`·`… IN SNOWFLAKE INTELLIGENCE …` 각 2행 확인. 실행 SQL 로그 = `09_AGENT_spec_구현.sql`.
> ⚠️ **스모크(§3) 방법 제약**: `cortex_agent_query`(SNOWFLAKE.CORTEX.DATA_AGENT_RUN)는 **트라이얼 계정에서 차단**('Access denied for trial accounts') → 프로그램적 회귀 불가. **CoWork UI(https://ai.snowflake.com)에서 §3 문항을 수동 질의**하여 판정(⬜) 채움.

## 0. 연결 아키텍처 (advanced 경로)

계정에 **명시적 SI object**(`SNOWFLAKE_INTELLIGENCE_OBJECT_DEFAULT`, 02 §F 생성)가 존재 → 자동 노출이 아니라 **`ALTER SNOWFLAKE INTELLIGENCE … ADD AGENT`** 로 명시 등록해야 CoWork에 노출된다.

```
GN_DW.SERVING.AGENT_MEMBER  ─┐
                              ├─ ADD AGENT ─► SNOWFLAKE_INTELLIGENCE_OBJECT_DEFAULT ─► CoWork(https://ai.snowflake.com)
GN_DW.SERVING.AGENT_OVERALL ─┘                    (object USAGE = 소비 3역할, 02 §F)
```

- 소비자는 **자기 default role**로 CoWork에서 Agent 실행 → 역할에 ① Agent USAGE(08 §4.3) ② SI object USAGE(02 §F) ③ SV REFERENCES,SELECT + GOLD SELECT(05·02) ④ ANALYTICS_WH USAGE + default warehouse 필요.
- Cortex 사용권(SNOWFLAKE.CORTEX_USER)은 PUBLIC 상속.

## 1. 선행 배포 (사용자 = GN_DW_ADMIN) — 08 §4

> 미완료 시 §2 진행 불가. 완료 확인: `SHOW AGENTS IN SCHEMA GN_DW.SERVING;` 가 2행.

1. **save**(semantic_studio): `cortex_agent_save` — `AGENT_MEMBER`·`AGENT_OVERALL` (file_path=`cortex_project/*.agent.yaml`, fqn=`GN_DW.SERVING.*`).
2. **publish**: `SHOW VERSIONS IN AGENT GN_DW.SERVING.AGENT_MEMBER;` 로 미게시 버전 확인 후 `cortex_agent_publish`(각 Agent).
3. **USAGE grant**(08 §4.3): `AGENT_MEMBER`·`AGENT_OVERALL` → `GN_DW_ANALYST`·`GN_DW_VIEWER`·`GN_DW_SERVICE`.

**확인 SQL**
```sql
SHOW AGENTS IN SCHEMA GN_DW.SERVING;                       -- 2행
SHOW GRANTS ON AGENT GN_DW.SERVING.AGENT_MEMBER;           -- USAGE: ANALYST/VIEWER/SERVICE
SHOW GRANTS ON AGENT GN_DW.SERVING.AGENT_OVERALL;
```

## 4. VQR 등록 (정확도 1순위 · 06 §3, 병행 권장)

> Agent 정확도의 근본은 SV 스티어링. 06 §3 SV별 검증쿼리를 verified query로 등록하면 유사 질문의 SQL 생성이 안정화된다.

- semantic_studio `semantic_view/vqr_management` 또는 SV YAML의 `verified_queries`(AI_VERIFIED_QUERIES)로 SV별 대표 질의 등록.
- 우선순위: 납부율(연/월 스코프)·회원구분별 회비·전이유형별·채널별 발송·행사종류별·예산구분별(각 07 gold SQL과 동일).
