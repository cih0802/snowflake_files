# 7단계 — 거버넌스·운영 (GN_DW Cortex Agent/SV)

> 배포된 2 Agent(회원·overall)와 5 SV의 **사용량 가시성 → 비용 통제 → 품질 폐루프**를 운영 체계로 고정한다.
> 근거: `08_mornitoring/02_Cortex_AI_사용량 모니터링.md`·`03_비용제한_예산 쿼터.md`·`04_알림_및 거버넌스.md`. 뷰명·구문은 해당 문서에서 확정된 것만 사용(실행 전 `DESCRIBE VIEW`로 컬럼 확인).
> **핵심 제약**: 서버리스 소비는 **Resource Monitor로 차단 불가** → 하드 차단은 **Per-user quota(Preview) 또는 RBAC**뿐. Budget은 알림 전용.

---

## 0. GN_DW AI 소비 과금 매핑 (무엇이 크레딧을 쓰나)

| 소비 경로 | service_type | 과금 | GN_DW 대상 |
|---|---|---|---|
| Cortex Agent Orchestrator | `CORTEX_AGENTS` | 크레딧 | `AGENT_MEMBER`·`AGENT_OVERALL` 실행 |
| Cortex Analyst (text-to-SQL) | `AI_SERVICES` | 크레딧 | Agent 도구 5 SV의 NL→SQL |
| Snowflake Intelligence(CoWork) | `SNOWFLAKE_INTELLIGENCE` | 크레딧 | CoWork 챗 세션 |
| Cortex Code(CoCo) | `CORTEX_CODE_SNOWSIGHT`/`_CLI` | 크레딧 | 개발/운영(본 프로젝트 작업) |
| **웨어하우스 실행** | (WH 크레딧) | 크레딧 | **`GN_DW_ANALYTICS_WH`** = SV SQL 실제 실행 컴퓨트 |

> Agent 1회 질의 = `CORTEX_AGENTS`(오케스트레이션) + `AI_SERVICES`(Analyst) + `GN_DW_ANALYTICS_WH`(SQL 실행) 크레딧이 함께 발생. `AI_INFERENCE`(REST API)는 **달러 직과금**이라 metering 뷰에 없음(청구서 확인).

## 1. 가시성 — 사용량 모니터링 (정본 뷰, 중복 합산 금지)

> `SNOWFLAKE.ACCOUNT_USAGE.*` 접근 = ACCOUNTADMIN 또는 GOVERNANCE_VIEWER/OBJECT_VIEWER. 뷰는 최대 3시간+ 지연.

### 1-1. 계정 AI 크레딧 총량(30일)
```sql
SELECT service_type, SUM(credits_used) AS total_credits
FROM SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY
WHERE service_type IN ('AI_SERVICES','CORTEX_AGENTS','CORTEX_CODE_CLI','CORTEX_CODE_SNOWSIGHT','SNOWFLAKE_INTELLIGENCE')
  AND usage_date >= DATEADD('day', -30, CURRENT_DATE())
GROUP BY service_type ORDER BY total_credits DESC;
```

### 1-2. Agent별 사용량(정본: `CORTEX_AGENT_USAGE_HISTORY`)
```sql
SELECT * FROM SNOWFLAKE.ACCOUNT_USAGE.CORTEX_AGENT_USAGE_HISTORY
WHERE start_time >= DATEADD('day', -30, CURRENT_DATE());   -- AGENT_MEMBER/AGENT_OVERALL 필터는 DESCRIBE 후 컬럼 확인
```

### 1-3. Analyst 사용자별 요청·크레딧(정본: `CORTEX_ANALYST_USAGE_HISTORY`)
```sql
SELECT username, SUM(credits) AS credits, COUNT(*) AS request_cnt
FROM SNOWFLAKE.ACCOUNT_USAGE.CORTEX_ANALYST_USAGE_HISTORY
WHERE start_time >= DATEADD('day', -30, CURRENT_DATE())
GROUP BY username ORDER BY credits DESC;
```

### 1-4. CoWork(SI) 사용량 + 실행 WH 크레딧
```sql
-- CoWork
SELECT * FROM SNOWFLAKE.ACCOUNT_USAGE.SNOWFLAKE_INTELLIGENCE_USAGE_HISTORY
WHERE start_time >= DATEADD('day', -30, CURRENT_DATE());
-- SV SQL 실행 컴퓨트(GN_DW_ANALYTICS_WH)
SELECT TO_DATE(start_time) AS d, SUM(credits_used) AS wh_credits
FROM SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY
WHERE warehouse_name = 'GN_DW_ANALYTICS_WH' AND start_time >= DATEADD('day', -30, CURRENT_DATE())
GROUP BY 1 ORDER BY 1;
```

> 정본 뷰만 사용(중복 합산 금지). 기능별 뷰 합산은 `METERING`의 `AI_SERVICES` 총합과 정확히 일치하지 않는 것이 정상(부분집합).

## 2. 소프트 제어 — Budget (알림, 차단 아님)

- **계정 레벨 Budget 활성화** + 월 한도·알림 설정(Snowsight **Admin » Cost Management » Budgets** UI 권장 — DDL은 버전차).
- 서버리스(Cortex) 개별 지정 범위는 Edition/버전차 → **계정 레벨 Budget으로 전체 AI 지출 모니터링**이 가장 안전.
- 한계: Budget은 **자동 차단 없음**. 임계 도달 시 알림만 → 운영자 후속조치(차단/회수) 필요.

```sql
-- 참고(UI 권장): 커스텀 Budget
CREATE OR REPLACE SNOWFLAKE.CORE.BUDGET gn_dw_ai_budget();
CALL gn_dw_ai_budget!SET_SPENDING_LIMIT(500);              -- 월 크레딧 한도(예)
CALL gn_dw_ai_budget!SET_EMAIL_NOTIFICATIONS('["admin@goodneighbors.org"]');
```

## 3. 하드 제어 — RBAC & Per-user quota

### 3-1. RBAC (이미 구축 + 점검)
- Agent USAGE는 소비 3역할(`GN_DW_ANALYST`·`GN_DW_VIEWER`·`GN_DW_SERVICE`)에만 부여 완료(08 §4·09). 비인가 역할은 CoWork에서 Agent 미노출.
- `SNOWFLAKE.CORTEX_USER`가 `PUBLIC`에 개방됐는지 점검 → 통제 강화 시 회수 후 전용 역할로만 부여(조직 영향 크므로 공지 후).
```sql
SHOW GRANTS ON AGENT GN_DW.SERVING.AGENT_MEMBER;   -- USAGE = ANALYST/VIEWER/SERVICE 확인
SHOW GRANTS OF DATABASE ROLE SNOWFLAKE.CORTEX_USER; -- PUBLIC 개방 여부 점검
```

### 3-2. Per-user quota (Preview · 유일한 개인 하드 차단)
- 월/일 사용자별 크레딧 한도 → 초과 시 자동 차단, 리셋/상향 시 해제. 태그로 팀 단위 적용.
- **Preview** → 계정 가용성 확인 후 도입, DDL은 공식 문서(`budgets/per-user-quotas`) 최신본 참조.
- ⚠️ `ALTER USER … SET RESOURCE_MONITOR_CREDIT_QUOTA` 구문은 **존재하지 않음**.

## 4. 실시간 경고 — Alert (2~6시간 주기)

```sql
CREATE OR REPLACE NOTIFICATION INTEGRATION gn_admin_email_int
  TYPE = EMAIL ENABLED = TRUE ALLOWED_RECIPIENTS = ('admin@goodneighbors.org');

CREATE OR REPLACE ALERT gn_cortex_cost_alert
  WAREHOUSE = GN_DW_ANALYTICS_WH
  SCHEDULE = 'USING CRON 0 */2 * * * Asia/Seoul'
  IF (EXISTS (
    SELECT 1 FROM SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY
    WHERE service_type IN ('AI_SERVICES','CORTEX_AGENTS','SNOWFLAKE_INTELLIGENCE')
      AND usage_date = CURRENT_DATE()
    GROUP BY usage_date HAVING SUM(credits_used) > 50 ))
  THEN CALL SYSTEM$SEND_EMAIL('gn_admin_email_int','admin@goodneighbors.org',
       'GN_DW Cortex AI 비용 경고','일일 AI 크레딧이 임계치(50)를 초과했습니다.');
ALTER ALERT gn_cortex_cost_alert RESUME;
```
> `METERING_DAILY_HISTORY` 지연 → "사후 경고" 성격. 즉시 통제는 Per-user quota 병행.

## 5. 품질 폐루프 (정확도 운영 — Agent 고유)

> 비용뿐 아니라 **답변 정확도**를 지속 관리한다. 소비 로그·오답을 스펙 개선으로 되먹임.

| 신호 | 출처 | 액션 |
|---|---|---|
| 오답/라우팅 오류 | CoWork 사용자 피드백·10 §3 스모크 FAIL | 08 tool `description`/`orchestration` 보강 → `cortex_agent_write`→save→publish 재배포 |
| 비활성 지표 임의 산출 | 가드레일 ⓖ 위반 | 08 `instructions.system` 비활성 목록 강화 |
| 반복 질문 패턴 | `CORTEX_ANALYST_USAGE_HISTORY`·질의 로그 | 대표 질의를 **VQR(verified query)** 등록(06 §3) → 정확도 스티어링 |
| 요청 실패/지연 | AI Observability(`GET_AI_OBSERVABILITY_EVENTS`, paid) | 근본원인 분석 후 SV/Agent 수정 |

- **재배포 주의**: SV `CREATE OR REPLACE SEMANTIC VIEW`는 grant 삭제 → REFERENCES,SELECT 3역할 재부여(05 §6). Agent는 `cortex_agent_save`(in-place ALTER)로 grant 보존.
- **데이터 재적재**: GOLD 재적재 시 07 평가셋 ground truth 재생성 → 회귀 재실행.

## 7. 트라이얼 제약 & paid 이관

| 항목 | 트라이얼 | paid 이관 후 |
|---|---|---|
| 에이전트 실행(DATA_AGENT_RUN) | ❌ 차단 → 사용량 로그 미발생 | ✅ 스모크·회귀 + 사용량 뷰 채워짐 |

| Budget/Alert | 생성 가능하나 AI 소비 데이터 희박 | 실효 |

| ORGANIZATION_USAGE | 조직/Enterprise+ 한정 | 계정 등급 따라 |
