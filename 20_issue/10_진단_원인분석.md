# 10. 진단 · 원인분석 (업무단계: 진단)

> 이슈를 열기 전에 **왜 생기는가**를 규명하는 단계. SILVER 설계 중 축적된 재사용 진단패턴·교훈과, 트랙 착수 전 판정해야 할 구조적 게이트를 모읍니다.
> 전체 이슈 인덱스: `00_INDEX_이슈원장.md`

## 1. 재사용 진단 패턴 (P-시리즈)
> ⚠️ P1~P9는 **설계 원칙/진단 렌즈**이며 개별 이슈 번호가 아님.

| # | 증상 | 결론(진단 규칙) |
|---|---|---|
| P1 코드→라벨 | 코드그룹 "추정"·라벨 미상 | `(CD_ID, DTL_CD_ID)` **복합** 조인 필수 (DTL_CD_ID 전역 비유일) |
| P2 유사키 함정 | `_NO`/`_ID` 같은 것인지 불확실 | `COUNT(DISTINCT)` 비교 → 자릿수 다르면 grain 다름 |
| P3 병합 fan-out | JOIN 행 폭발 우려 | 중복키=0 & 조인후≈base → N:1 LEFT JOIN(스파인 보존) |
| P4 SUM 중복 | PK 있어도 합계 부풀 | 동일 실물키 GROUP BY → measure별 grain 분리 |
| P5 미검증 스키마 | "(LLM생성)" 표기 | 정의서 전까지 키·관계 단정 금지 |
| P6 실측 핸드오프 | 실데이터 타 어카운트 | 쿼리팩+회수 템플릿 분리, 해석은 설계자 |

## 2. 핵심 교훈 (근본원인 예방)
- 사전의 **"추정"·"(LLM생성)"은 실측 전 사실 아님** → 단정 금지(P5).
- `_NO`/`_ID` 유사 키는 grain부터 의심 → distinct 비교(P2).
- PK 있어도 measure는 실물 중복 별도 확인(P4).
- **정의서 타입 ≠ 실측 타입**: UNION/캐스팅 전 `INFORMATION_SCHEMA.COLUMNS` 실타입 확인.
  - 예: `SNDNG_STDR_DE`·`SEND_DATE` 문서상 TEXT였으나 실측 DATE → `::TIMESTAMP_NTZ` 필요.
- **다계열 UNION 키는 판별자를 PK에 포함**(예 `SEND_CHANNEL`) → 미래 키 충돌 방지.

## 3. 설계 노트 (개선 후보·비긴급)
- **월 팩트 conform 갭**: `FACT_MEMBER_MONTHLY`·`ERP_BUDGET`는 `MONTH_KEY`(YYYYMM) grain인데 **DIM_MONTH 부재**, `DIM_DATE`는 일 grain → 월 팩트가 날짜차원에 단일키 조인 불가. **월 차원 신설 또는 degenerate 명시** 검토.
- **`DW_BATCH_ID` 전 SILVER NULL**: `NULL AS DW_BATCH_ID` 하드코딩(dbt `invocation_id` 미주입). 감사메타 보완 필요(GOLD는 `gold_meta` 매크로로 정상 주입). *내부 처리·현업 판정 대상 아님.*
 - **🔴 FROZEN(삭제 금지) — D1 후속**: 스캐폴드 팩트 6종(FGA·FSE·FMM·FME·FEP·FTD)은 `incremental + append + pre-hook TRUNCATE + full_refresh:false`(순서9 G-1/G-2 개정 — `table`은 CTAS가 06_DDL FK/타입 드롭해 폐기). 매 run TRUNCATE 후 전량 재적재 → 잔존행 0·멱등이나 **대량시 재적재 비용 발생**. **마감 전 볼륨·비용 재평가 후 진성 `incremental`(+merge, unique_key에 활성 SK) 재전환 여부 결정 필수.** (의사결정 상세: 문서30 D1)

## 5. [순서9-C 2026-07-15] 진단 결론 · 신규 교훈

### 5-A. 이슈 E (EVENT_KEY 고아 23%) 실데이터 진단 → **마스터 누락(외부)** 확정
- 고아 263,611 중 **99.98%가 `TD_MS_EVENT_PRTCPNT_DTL`**(행사참여상세). CRMN참여는 정상(50).
- 고아 참여일 2020–2026 = 정상분과 **동일 기간**(마스터 기간밖 아님).
- 고아 키에 **정상과 동일한 16자 `CR` 형식** 다수 + 소수 비정형 → 지배요인 = **행사 마스터(`CRM_EVENT`) 추출 불완전**.
- **결론**: 내부 수정 불가. 최종 해소 = 행사 마스터 전량입고(외부). 현재 `severity:warn` 유지 정당. (현업엔 "키체계 아닌 마스터 추출범위 문제"로 회신 유도 — 문서20 §C-E)

### 5-B. 데이터로 확정한 설계결정
- **A-2/Q9 `_SOURCE_SYSTEM`**: `AGENCY_AD_PERFORMANCE.SOURCE_SYSTEM`={DIGITAL·VIDEO·REBROADCAST}뿐(GoogleAds 없음=GA4쪽) → GOLD `DW_SOURCE_SYSTEM='AGENCY'` 상수 + 매체구분은 속성 보존. **내부 확정**.

### 5-C. 신규 재발방지 교훈 (P-시리즈 보강)
- **P7 merge 차원 stale**: GOLD dim = `incremental merge`(pre-hook TRUNCATE 없음) → **상류 grain/로직 변경 시 옛 행 잔존**. 로직 변경 시 대상 `TRUNCATE` 후 재적재 필수. (DIM_GA_EVENT 2,842→2,846 사고)
- **P8 test 실행 검증**: 스키마 테스트는 **저작만으로 검증 아님** — `build`(run+test) 실행해야 잠재버그 표면화. `run` 단독 금지(0행 회귀 통과). (153-test suite 최초 실행서 17 ERROR 표면화)
- **P9 다운스트림 추적**: 상류 모델 grain 변경 전 소비 모델의 키 의존 확인. (GA4_EVENT_DIM 축약이 DIM_GA_EVENT 조합키를 파괴한 사고)
- **severity 정책(메달리온 BP)**: Silver 참조무결성=알려진 원천 미완전이면 `warn` 관측, `error` 는 구조 불변식(차원 PK)만. 마스터 입고 후 error 복귀(문서50 추적표).

### 5-D. [2026-07-15] identity 활성화 진단 (ID-활성)
- **enablement 전제 무효화**: `DIM_MEMBER_IDENTITY`의 `enabled=false` 사유("GA4_IDENTITY 비활성")는 SILVER `GA4_IDENTITY`(1,348행)·`IDENTITY_MEMBER_XREF`(1,348행)가 이미 적재돼 **무효** → 활성화 정당.
- **🔴 grain fan-out 진단**: `IDENTITY_MEMBER_XREF`는 `user_pseudo_id` 단위(회원 1명이 최대 3 pseudo·67명). `MEMBER_DK`로 순진 조인 시 차원 행 폭발→`IDENTITY_SK` 중복(PK 위반). **해소 = MEMBER_DK 집계 dedup 후 조인**(DIM·FACT 공통 적용). `unique(IDENTITY_SK)` 테스트 PASS로 검증.
- **⚠️ 커버리지 오인 금지**: 이벤트 단위 `user_id` 채움률 ~4.2%(1일 샤드). FACT 집계행 기준 79.6%는 **grain 효과**(미매칭=IDENTITY_SK 0 단일버킷)이지 트래픽 식별률 아님. 확정치로 쓰지 말 것. G-5 전기간 입고 시 재검증(문서50).
