# 04. 알림 및 거버넌스 전략

Cortex AI 비용·사용량에 대한 실시간 알림 구성과 운영 권장 전략을 정리한다.

## 1. Alert 기반 알림

`ACCOUNT_USAGE` metering 뷰를 주기적으로 검사해 임계치 초과 시 이메일 등으로 알린다.
Alert 자체도 서버리스로 실행되며 소량의 크레딧(`SERVERLESS_ALERTS`)을 소비한다.

```sql
-- 사전 준비: 이메일 notification integration (ACCOUNTADMIN)
CREATE OR REPLACE NOTIFICATION INTEGRATION admin_email_int
    TYPE = EMAIL
    ENABLED = TRUE
    ALLOWED_RECIPIENTS = ('admin@company.com');  -- 검증된 이메일만 허용
```

```sql
-- 일일 Cortex AI 크레딧이 임계치 초과 시 알림
CREATE OR REPLACE ALERT cortex_cost_alert
    WAREHOUSE = COMPUTE_WH
    SCHEDULE = 'USING CRON 0 */2 * * * Asia/Seoul'   -- 2시간마다
    IF (EXISTS (
        SELECT 1
        FROM SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY
        WHERE service_type IN (
                'AI_SERVICES', 'CORTEX_AGENTS',
                'CORTEX_CODE_CLI', 'CORTEX_CODE_SNOWSIGHT',
                'SNOWFLAKE_INTELLIGENCE'
              )
          AND usage_date = CURRENT_DATE()
        GROUP BY usage_date
        HAVING SUM(credits_used) > 50            -- 일 50크레딧 초과
    ))
    THEN
        CALL SYSTEM$SEND_EMAIL(
            'admin_email_int',
            'admin@company.com',
            'Cortex AI 비용 경고',
            'Cortex AI 일일 크레딧 사용량이 50을 초과했습니다.'
        );

-- 생성된 Alert는 기본적으로 일시중지 상태이므로 활성화 필요
ALTER ALERT cortex_cost_alert RESUME;
```

> **주의사항**
> - `METERING_DAILY_HISTORY`는 지연(최대 수 시간)이 있어, 실시간 차단 용도로는 부적합하고
>   "사후 경고" 성격이다. 즉각적인 통제가 목표라면 Per-user quota(하드 차단)를 병행한다.
> - `SYSTEM$SEND_EMAIL`의 인자는 (integration_name, recipients, subject, body) 순서다.
> - CRON 표현식의 타임존은 계정 파라미터가 아니라 문자열 내에서 지정한다.

## 2. 운영 권장 전략 (계층적 접근)

| 단계 | 수단 | 목적 |
|------|------|------|
| 가시성 | `METERING_DAILY_HISTORY` + `CORTEX_*_USAGE_HISTORY` 대시보드 | 누가·얼마나 쓰는지 파악 |
| 소프트 제어 | Budget + 이메일 알림 | 한도 접근 시 경고 |
| 실시간 경고 | Alert (2~6시간 주기) | 급증 감지 |
| 하드 제어(개인) | Per-user quota (Preview) | 사용자별 상한 초과 시 차단 |
| 하드 제어(접근) | RBAC (`CORTEX_USER` 역할 통제) | 비인가 사용자 원천 차단 |

## 3. 거버넌스 체크리스트

- [ ] `SNOWFLAKE.CORTEX_USER` 역할이 `PUBLIC`에 개방되어 있는지 점검 (필요 시 회수)
- [ ] Cortex 사용 전용 역할을 만들고 승인된 사용자에게만 부여
- [ ] 계정 레벨 Budget 활성화 및 월 한도·알림 설정
- [ ] Per-user quota 사용 가능 여부 확인 후, 고위험 사용자군에 적용 검토
- [ ] AI 사용량 대시보드(주간/월간)를 정기 리뷰 루틴에 포함
- [ ] `AI_INFERENCE`(REST API, 달러 과금)는 metering 뷰에 없으므로 청구서로 별도 확인
- [ ] 각 뷰의 정본(canonical) 여부를 확인해 중복 합산 방지

## 4. 한계 및 유의점

- 서버리스 소비는 **Resource Monitor로 차단할 수 없다.** 하드 차단은 Per-user quota 또는 RBAC뿐이다.
- Budget은 알림 전용이며 자동 지출 중단 기능이 없다.
- `ACCOUNT_USAGE` 뷰는 지연이 있어 "지금 이 순간"의 비용을 실시간 반영하지 못한다.
- Per-user quota는 Preview이므로 GA 이전에는 기능/구문이 변경될 수 있다.
- 기능별 사용량 뷰는 metering 총합과 정확히 일치하지 않는 것이 정상이다(부분집합).
